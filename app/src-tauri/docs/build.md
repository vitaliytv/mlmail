## Огляд

Описує поведінку з читанням публічних OAuth client IDs під час запуску, щоб значення були доступні для застосунку без ручного дублювання в коді. Джерело конфігурації залишається зовнішнім, тому поведінка залежить від наявних значень у середовищі виконання.

## Поведінка

1. Під час збірки вбудовує значення публічних OAuth client IDs у бінарник, щоб встановлена програма могла стартувати без `.env` у поточній робочій теці.
2. Дозволяє `dotenv.secret` перевизначити ці значення під час виконання.
3. Перечитує середовище для кожного з підтримуваних ключів окремо і підхоплює лише ті, що задані.
4. Повідомляє збірці, що ці значення можуть змінитися, щоб артефакт перескладався після оновлення середовища.
5. Також відстежує зміни в `.env`, щоб збірка залишалася синхронізованою з конфігурацією.

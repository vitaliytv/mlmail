## Огляд

`EncryptedPrefs` — read-only обгортка для доступу до захищених значень у застосунку. Підтримує публічні точки входу `EncryptedPrefs` і `new`, працює fail-safe: перехоплює помилки, не кидає винятки назовні та за певних збоїв повертає порожнє значення, наприклад `null`, замість помилки. Не виконує запис у ФС чи БД.

## Поведінка

- EncryptedPrefs — зберігає, читає й очищає сесію через Android-сторедж, перехоплюючи помилки й повертаючи безпечний результат замість падіння.
- new — створює обгортку для роботи з Android-стореджем у межах поточного застосунку.

## Публічний API

- EncryptedPrefs — надає доступ до налаштувань у read-only режимі та поводиться fail-safe: не кидає винятків назовні
- new — створює новий екземпляр; за певних помилок повертає порожнє значення, наприклад `null`, замість винятку

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.

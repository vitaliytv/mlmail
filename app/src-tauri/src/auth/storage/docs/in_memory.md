## Огляд

`InMemoryStorage` і `new` дають тимчасове сховище стану сесії в пам’яті без запису у ФС чи БД. Воно призначене для сценаріїв, де потрібен порожній старт, читання, оновлення й очищення даних у межах одного процесу. Компонент працює fail-safe: помилки не виходять назовні, а в окремих випадках повертається порожнє значення, наприклад `null`, замість винятку.

## Поведінка

- InMemoryStorage — зберігає один поточний стан сесії в пам’яті для читання, оновлення й очищення; не пише у ФС чи БД; у разі помилок повертає помилку без падіння.
- new — створює порожнє in-memory сховище без збереженої сесії.

## Публічний API

- InMemoryStorage — read-only сховище в оперативній пам’яті без запису у ФС чи БД; при помилках поводиться fail-safe, не виносить винятки назовні й за потреби повертає порожнє значення замість помилки
- new — створює сховище з таким самим read-only і fail-safe поводженням; у разі помилки не кидає виняток назовні й може повернути порожнє значення

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.

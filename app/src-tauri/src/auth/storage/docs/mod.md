## Огляд

Файл визначає `StoredSession`, `RefreshTokenStorage`, `SharedStorage` і `platform_storage` як контракт і платформне сховище refresh token для відновлення авторизації користувача за збереженою сесією. Він потрібен, щоб різні частини застосунку працювали зі спільним сховищем сесії через єдиний інтерфейс без запису у ФС чи БД. За певних помилок читання сховище повертає порожнє значення (`null`) замість винятку.

## Поведінка

- `StoredSession` представляє збережену сесію користувача з email і refresh token для подальшого відновлення авторизації.
- `RefreshTokenStorage` задає спільний контракт для сховищ refresh token: збереження, завантаження та очищення сесії; за певних помилок завантаження може повертати порожнє значення замість винятку.
- `SharedStorage` надає спільне посилання на реалізацію сховища, щоб різні частини застосунку працювали з одним контрактом зберігання.
- `platform_storage` на macOS створює файлове сховище refresh token для платформи.
- `platform_storage` на Android створює платформне зашифроване сховище refresh token.

## Публічний API

- StoredSession — описує прочитану сесію користувача з авторизаційними даними й строками їхньої дії.
- RefreshTokenStorage — надає доступ до читання refresh token для повторної авторизації без запису в сховище.
- SharedStorage — надає read-only доступ до спільних авторизаційних даних між частинами застосунку.
- platform_storage — повертає платформозалежний доступ до авторизаційних даних без запису в сховище.
- platform_storage — повторно експортований публічний символ для сумісності API; має ту саму поведінку read-only доступу.
- За певних помилок читання API повертає порожнє значення (`null`) замість винятку.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.

## Огляд

Описує два контрактні типи помилок: `AuthError` для збоїв автентифікації та взаємодії з мережею, а також `StorageError` для помилок сховища. Файл існує, щоб ці збої можна було однаково передавати між частинами системи.

## Поведінка

- AuthError — описує помилки автентифікації та OAuth для передачі у UI/JSON: скасування входу, мережеві та платформні збої, проблеми конфігурації, storage і потребу повторної автентифікації.
- StorageError — описує помилки backend-підсистеми сховища, які можуть бути перетворені в помилку автентифікації.

## Публічний API

- `AuthError` — публічна функція для позначення помилок автентифікації й доступу під час роботи через мережу; зручно використовувати як явний `error-marker` для цих сценаріїв.
- `StorageError` — публічна функція для позначення помилок читання або перевірки даних у read-only сценаріях; не пов’язана із записом у ФС чи БД і може слугувати `error-marker` для сховища або конфігурації.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).

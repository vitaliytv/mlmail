## Огляд

Файл надає публічну функцію `extract_email`, яка повертає email, якщо його вдається коректно визначити з вхідного значення. Модуль працює в режимі read-only, не виконує записів і перехоплює помилки всередині себе, тому не викидає винятків назовні. Якщо email неможливо отримати через некоректні або неповні дані, функція повертає порожнє значення, наприклад `null`.

## Поведінка

1. `extract_email` намагається дістати email із `id_token`, щоб отримати ідентифікатор користувача для подальшої автентифікаційної логіки.
2. `extract_email` очікує токен у форматі JWT і бере з нього інформаційну частину з даними користувача.
3. `extract_email` декодує цю частину та читає її як JSON-обʼєкт, щоб знайти поле `email`.
4. `extract_email` повертає значення email, якщо поле `email` присутнє та має текстовий формат.
5. `extract_email` повертає порожнє значення, якщо токен не схожий на JWT, якщо інформаційну частину неможливо декодувати, якщо вміст не є коректним JSON або якщо поле `email` відсутнє.
6. `extract_email` працює у fail-safe режимі: не пише у ФС чи БД, не кидає винятків назовні та за помилкових вхідних даних повертає порожнє значення.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.

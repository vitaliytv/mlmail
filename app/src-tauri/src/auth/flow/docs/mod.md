## Огляд

Файл звіряє поточну операційну систему з двома підтриманими значеннями — `macos` і `android` — та обирає відповідний сценарій автентифікації лише для однієї з цих платформ. Якщо значення платформи не збігається ні з `macos`, ні з `android`, файл не активує жоден із цих двох сценаріїв і залишає інші платформи поза цією реалізацією.

## Поведінка

1. Визначає платформу виконання й підключає лише той варіант auth flow, який відповідає поточній операційній системі.
2. Для macOS робить доступним окремий platform-specific сценарій автентифікації, щоб застосунок використовував нативну поведінку цієї платформи.
3. Для Android робить доступним окремий platform-specific сценарій автентифікації, щоб застосунок використовував нативну поведінку цієї платформи.
4. На інших платформах не додає жодного з цих сценаріїв, бо файл описує лише macOS- і Android-гілки.
5. Працює в режимі read-only: не змінює файлову систему та не записує дані в базу.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).

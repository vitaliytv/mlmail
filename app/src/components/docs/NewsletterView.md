Компонент показує вміст листа в одному з трьох режимів: newsletter-шаблон, резюме або переклад. Також він дає поставити запитання про лист, створити або змінити шаблон і позначити лист як завдання.

## Поведінка

- Після отримання нового листа скидає попередні результати, підвантажує шаблони та вибирає активний шаблон для цього відправника.
- Якщо для листа знайдено newsletter-шаблон:
  - намагається витягти статті за інструкцією шаблону;
  - показує список знайдених матеріалів із посиланнями та описами;
  - у разі помилки показує повідомлення про недоступність локальної моделі.
- Якщо шаблон не знайдено:
  - у режимі резюме показує короткий текстовий підсумок листа;
  - у режимі перекладу показує HTML-переклад у sandboxed iframe;
  - для порожнього листа показує нейтральний стан без повторних спроб.
- Успішні результати резюме та перекладу кешуються для конкретного листа, щоб повторне відкриття не запускало повторну обробку.
- Під час завантаження, резюмування, перекладу або рендеру показує індикатор очікування.
- Запитання до листа надсилається лише якщо введено непорожній текст; при успіху відповідь показується в окремому блоці, при помилці — повідомлення про збій.
- Дії з шаблонами:
  - створює новий шаблон на основі відправника поточного листа;
  - відкриває поточний шаблон для редагування;
  - зберігає шаблон і оновлює відображення після успіху;
  - видаляє активний шаблон і повертається до fallback-режиму.
- Якщо створений або змінений шаблон належить до типу завдань, запускає повторний скан листів для позначення відповідних повідомлень.
- Дає вручну позначити поточний лист як завдання через окрему експортовану дію.

## Гарантії поведінки

- Для одного листа повторно не виконує дорогі LLM-операції, якщо вже є успішний результат у кеші.
- Невдала обробка не блокує повторну спробу.
- Шаблонний режим має пріоритет над резюме та перекладом.
- Переклад відображається тільки як вбудований HTML-документ із обмеженим доступом.
- Кнопка збереження шаблону недоступна, доки не задано мінімальні умови для валідного шаблону.
- Порожній запит не надсилається.
- При зміні листа старі відповіді, переклад і стан помилок очищаються.

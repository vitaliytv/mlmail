## Огляд

Файл надає `useNewsletterRender` для отримання результату рендерингу newsletter з тексту листа. Він потрібен, щоб безпечно обробляти вміст листа у read-only сценарії без запису у ФС чи БД. Функція працює у fail-safe режимі: перехоплює помилки і не кидає винятків назовні. За певних помилок або непридатного вхідного чи вихідного результату повертає порожнє значення (`null`) замість винятку.

## Поведінка

1. `useNewsletterRender` повертає інтерфейс рендерингу newsletter, який перетворює вміст листа на структурований список матеріалів для подальшого використання в застосунку.

2. Якщо текст листа відсутній або містить лише порожні символи, `useNewsletterRender` завершує обробку без звернення до зовнішньої AI-моделі й повертає порожній результат. Це запобігає зайвій генерації там, де немає контенту для аналізу.

3. Перед генерацією `useNewsletterRender` підтягує актуальне AI-конфігурування середовища, щоб рендеринг виконувався з поточними налаштуваннями моделі та доступу.

4. Для побудови результату `useNewsletterRender` надсилає до AI-моделі інструкцію шаблону разом із текстом листа, щоб отримати уніфіковане представлення змісту у вигляді добірки статей.

5. Якщо AI-модель повертає відповідь у форматі, придатному для машинного читання, `useNewsletterRender` інтерпретує її як структурований список матеріалів і повертає цей список як результат рендерингу.

6. Якщо відповідь моделі або будь-який етап обробки виявляється непридатним для коректного розбору, `useNewsletterRender` працює у fail-safe режимі: не викидає виняток назовні та повертає порожнє значення замість аварійного завершення.

7. `useNewsletterRender` працює у read-only режимі: не записує дані у файлову систему чи базу даних, а лише формує результат на основі вхідного листа та відповіді зовнішнього AI-сервісу.

## Публічний API

- useNewsletterRender — формує текст випуску за шаблоном newsletter і повертає добірку статей із назвою, посиланням та описом; якщо згенерований результат не вдається розібрати, повертає null.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.

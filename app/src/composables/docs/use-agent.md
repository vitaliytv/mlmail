## Огляд

Піднімає `useAgent` як єдину точку доступу до готового agent-інтерфейсу для компонентів. Дає змогу читати стан і взаємодіяти з agent-потоком без додаткового дублювання логіки на рівні вищих шарів. Read-only: не виконує записів і не змінює зовнішній стан.

## Поведінка

1. `useAgent` створює готовий до роботи in-app agent для цього застосунку.
2. Він поєднує спільний agent із доменним контекстом mlmail, щоб взаємодія була прив’язана до Gmail-сценаріїв цього продукту.
3. Він повертає стан і керування для роботи з agent: налаштування підключення, журнал дій, запит, відповідь і підтвердження.
4. Він не виконує записів у файлову систему чи базу даних.
5. Він не бере на себе Tauri-специфіку; вона вже інкапсульована нижче в базовому composable.

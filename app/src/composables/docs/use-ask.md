## Огляд

Надає UI спосіб отримати коротку відповідь про конкретний email на основі його вмісту та запиту користувача. Працює read-only, не змінює ФС чи БД. Містить fail-safe обробку помилок: не пропускає винятки назовні й за окремих помилок повертає порожнє значення, зокрема `null`.

## Поведінка

1. `useAsk` надає інструмент для отримання стислої відповіді на запитання про конкретний email.
2. Приймає поточний стан листа та запит користувача, формує контекст відповіді на основі вмісту листа і передає його в модель разом із запитанням.
3. Під час обробки вмикає стан виконання, щоб UI міг показати, що триває запит до AI.
4. Якщо запит порожній або містить лише пробіли, одразу повертає порожнє значення без звернення до моделі.
5. Виконує fail-safe обробку: будь-яка помилка всередині процесу не виходить назовні, а результатом стає порожнє значення.
6. Після завершення запиту, незалежно від успіху чи помилки, скидає стан виконання назад.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.

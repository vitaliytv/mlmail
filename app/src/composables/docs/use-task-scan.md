## Огляд

`useTaskScan` надає поведінку сканування задач у read-only режимі без запису у файлову систему чи базу даних.

Модуль призначений для безпечного виконання цього процесу в інтерфейсному сценарії: він перехоплює помилки під час роботи та не кидає винятків назовні, щоб збої оброблялися fail-safe на межі виклику.

## Поведінка

1. `useTaskScan` надає спільний для всіх екранів стан списку листів, позначених міткою «Задача», щоб панель задач і дії з окремими листами відображали один і той самий актуальний результат.

2. Під час оновлення `useTaskScan` перечитує весь поточний вміст мітки «Задача» в Gmail і повністю замінює локальний список, щоб інтерфейс показував фактичний набір задач.

3. Якщо перечитування списку не вдається, `useTaskScan` безпечно очищає локальний список і не викидає помилку назовні, щоб збій інтеграції не ламав інтерфейс.

4. Під час сканування `useTaskScan` бере лише шаблони типу task, позначає початок процесу, обнуляє прогрес і фіксує загальну кількість шаблонів для відображення стану обробки.

5. Для кожного task-шаблону `useTaskScan` шукає листи, що відповідають його умовам, і додає мітку «Задача» кожному знайденому листу, щоб автоматично підняти нові задачі з вхідної пошти.

6. Якщо окремий шаблон не вдається обробити, `useTaskScan` ігнорує цю помилку, продовжує решту сканування та все одно просуває лічильник виконання, щоб часткові збої не зупиняли весь процес.

7. Після завершення сканування `useTaskScan` повторно перечитує список задач і знімає стан обробки, щоб інтерфейс показував підсумковий результат.

8. Для ручного позначення листа як задачі `useTaskScan` додає йому мітку «Задача» і одразу оновлює список, щоб нова задача з’явилася в панелі без додаткових дій користувача.

9. Для позначення задачі як виконаної `useTaskScan` знімає мітку «Задача» і відразу прибирає цей лист із локального списку, щоб завершена задача зникла з панелі без повного перезавантаження.

10. `useTaskScan` також тримає похідний підсумок кількості поточних задач, щоб інтерфейс міг показувати актуальний лічильник без окремого підрахунку.

## Публічний API

- useTaskScan — тримає спільний singleton-стан сканування задач, щоб будь-який екран міг позначити потребу в оновленні або запустити refresh, а панель задач одразу показала ці зміни.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).

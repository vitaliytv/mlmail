## Огляд

Модуль формує аналізний prompt для одного журналу виклику агента і надає публічні точки входу `buildAnalysisPrompt` та `useCallAnalysis` для запуску цього аналізу. Результат потрібен, щоб оцінювати ефективність дій агента й отримувати пропозиції змін у коді проєкту. Робота read-only: модуль не записує дані у ФС чи БД.

## Поведінка

- `buildAnalysisPrompt` — формує суцільний текстовий prompt із журналу одного виклику агента: запит, дії та відповідь, щоб передати це моделі для аналізу ефективності й пропозиції змін у коді проєкту.
- `useCallAnalysis` — надає два способи запустити аналіз журналу: через локальну модель у застосунку або через `pi`-CLI на хмарній моделі; читання даних, без запису у ФС чи БД.

## Публічний API

- buildAnalysisPrompt — збирає з журналу виклику самодостатній запит до моделі: показує request, actions і response та просить запропонувати зміни в коді, які зроблять такий виклик ефективнішим
- useCallAnalysis — запускає аналіз одного запису журналу двома шляхами: через локальну omlx-модель по direct HTTP або через `pi` CLI на cloud-моделі, яку піднімає Rust backend у debug-збірках

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).

## Огляд

Модуль надає публічну функцію `usePattern`, яка визначає шаблон теми листа, щоб працювати зі сталою формою схожих тем без прив’язки до змінних фрагментів. Модуль є read-only: не записує дані у ФС або БД. Усі внутрішні збої обробляються fail-safe: помилки перехоплюються і не викидаються назовні.

## Поведінка

1. `usePattern` надає helper для підказки стабільного шаблону теми листа, щоб із конкретної теми виділити повторювану частину, характерну для автоматичних повідомлень від відправника.

2. Під час запиту на підказку helper бере тему листа як базове значення та прибирає зайві пробіли по краях.

3. Якщо після цього тема порожня, helper одразу повертає порожній результат, бо шаблон визначати немає з чого.

4. Якщо тема непорожня, helper звертається до локальної LLM-конфігурації застосунку та просить on-device модель виділити лише сталий початок теми без змінної частини на кшталт версій, ідентифікаторів, дат чи імен.

5. Відповідь моделі використовується як бізнес-придатна підказка для правила зіставлення схожих автоматичних листів, із нормалізацією результату до безпечного значення для інтерфейсу.

6. Якщо локальна модель недоступна, середовище не завантажилось, відповідь некоректна або стається будь-яка інша помилка, helper працює у fail-safe режимі: не кидає виняток назовні та повертає вихідну очищену тему як придатне резервне значення.

7. `usePattern` працює в режимі read-only: не записує дані у файлову систему чи базу даних.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).

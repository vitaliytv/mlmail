Цей компонент є головною сторінкою для взаємодії з електронною поштою в додатку. Він відображає поточний вибір листа, надаючи можливість переглянути його оригінал, а також забезпечує інтерфейс для виконання низки дій з цим листом та управління застосунком загалом.

## Поведінка

1. **Ініціалізація при запуску:**
    * Отримує версію застосунку та встановлює її в заголовку сторінки та в тайтлі браузера.
    * Ініціалізує стан аутентифікації через `auth.initialize`.
    * Реєструє обробник подій `message` на вікні для перехоплення URL-адрес, які надсилаються з зовнішніх джерел, та відкриває їх у застосунку.
2. **Оновлення заголовка:**
    * Постійно відстежує дані користувача (email) та кількість листів у вхідних.
    * Автоматично оновлює тайтл сторінки та системний тайтл застосунку відповідно до статусу авторизації та наявності даних листа.
3. **Відображення стану:**
    * Якщо користувач авторизований, відображає:
        * Повідомлення про помилки (наприклад, вхідні помилки).
        * Інтерфейс для перегляду поточного листа, який включає оригінальні метадані, відображення HTML-тіла (захоплюючи та вставляючи стилі та логіку перехоплення кліків у `iframe`) або текстового тіла.
        * Панель з об'єктом для перегляду новин.
        * Панель кнопок дій, що розташована внизу екрану, що дозволяє відписатися, зберегти, позначити як завдання, створити правило, видалити лист, завантажити випадковий лист, переглянути журнал, відкрити шаблони, агент або журнал запитів.
    * Якщо лист завантажується, відображає скелетон-завантажувача.
    * Якщо користувач не авторизований, відображає кнопку для входу через Google.
4. **Управління правилами (Pattern Dialog):**
    * При натисканні на кнопку "Правило" відкривається діалогове вікно.
    * Якщо в поточній пошті є дані, діалогове вікно автоматично заповнюється відправником та темою.
    * Викликається функція для генерації пропозиції теми за допомогою локальної LLM. Якщо пропозиція успішна і не скасована, поле теми оновлюється цією пропозицією.
    * Користувач може створити фільтр на основі заданих критеріїв або видалити всі листи, що відповідають критерію відправника.
5. **Інтерактивність HTML:**
    * Якщо тіло листа містить HTML, воно відображається в `iframe`. Для забезпечення коректного відображення стилів та відстеження кліків у зовнішньому контенті, у HTML вставляються спеціальні скрипти та стилі, які обгортають оригінальне вміст.

## Гарантії поведінки

* Поведінка застосунку динамічно змінюється залежно від статусу аутентифікації користувача.
* Всі дії, пов'язані з поштою (збереження, видалення, фільтрація), виконуються асинхронно через API.
* При відображенні HTML-контенту гарантується, що інтерфейс коректно відображає оригінальний вміст, застосовуючи необхідні мета-елементи для роботи скриптів та стилів.
* Взаємодія з зовнішніми URL-адресами, перехопленіми кліком у тілі листа, гарантовано спрямовується через внутрішній API застосунку.

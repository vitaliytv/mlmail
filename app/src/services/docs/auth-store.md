## Огляд

`useAuthStore` надає доступ до стану авторизації, а `_resetForTest` повертає цей стан до початкового для тестів. Стор працює fail-safe: перехоплює помилки й не пропускає винятки назовні.

## Поведінка

- `useAuthStore` — надає стан і дії для автентифікації Gmail, роботи з вхідними листами та фільтрами, сам відновлює стан після помилок і при мережевих збоях повторює запити замість пробросу винятків назовні.
- Дія `loadMessageById` завантажує конкретний лист за id (наприклад, обраний у панелі задач) і робить його поточним; для неавторизованого користувача або порожнього id нічого не робить.
- Дія `openAttachment` завантажує одне вкладення поточного листа й відкриває його програмою за замовчуванням ОС; без поточного листа або id вкладення нічого не робить, відстежує окремим станом, яке саме вкладення зараз відкривається.
- Дії `trashCurrent` і `trashByQuery` після успішного переміщення листа(ів) у кошик також оновлюють спільний список задач (`useTaskScan`), щоб щойно вилучені листи одразу зникали з панелі відкритих завдань.
- `_resetForTest` — скидає весь внутрішній стан стору до початкових значень для тестів.

## Публічний API

- useAuthStore — дає доступ до стану автентифікації та дій для входу й виходу користувача, а також до завантаження випадкового чи конкретного (за id) листа; працює fail-safe і не кидає винятків назовні
- _resetForTest — скидає стан автентифікації до чистого стану для ізольованих тестів; працює fail-safe і не кидає винятків назовні

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).

## Огляд

Файл визначає `dispatch` як публічну точку входу для прямих викликів інструментів з UI без agent-шару. `dispatch` приймає запит від інтерфейсу, зіставляє його з доступною дією у спільній tool surface і повертає результат цього виклику назад у UI. Модуль призначений для того, щоб інтерфейс запускав такі дії через один стабільний вхід замість окремих спеціалізованих викликів. Поведінка модуля є read-only: він не записує дані у ФС або БД.

## Поведінка

1. `dispatch` приймає запит на виклик інструмента з інтерфейсу застосунку й спрямовує його до відповідної дії в спільній поверхні інструментів.
2. `dispatch` забезпечує єдину точку входу для прямих UI-викликів без участі agent-шару, щоб інтерфейс працював з інструментами в узгодженому форматі.
3. `dispatch` передає запит через транспорт застосунку, щоб виконання відбувалося в середовищі Tauri.
4. `dispatch` повертає результат в уніфікованій бізнес-обгортці: успіх із корисним результатом або помилка з описом збою.
5. `dispatch` не виконує запис у файлову систему чи базу даних у межах власної поведінки, тому його роль обмежена маршрутизацією виклику та поверненням відповіді.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).

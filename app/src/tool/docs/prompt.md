## Огляд

Файл визначає `systemPrompt`, який формує system prompt для агента роботи з Gmail inbox. Секція фіксує призначення цього prompt як джерела правил і меж поведінки агента на рівні системної інструкції. Модуль є `Read-only: не пише (ФС/БД)`, тому його роль обмежена підготовкою тексту prompt без побічних змін у сховищах.

## Поведінка

1. `systemPrompt` задає базову роль агента як помічника для роботи з Gmail inbox користувача, щоб усі подальші дії були обмежені пошуком листів, читанням листа повністю та переміщенням листів у Trash.

2. `systemPrompt` вимагає використовувати лише надані інструменти, щоб агент працював через контрольований tool surface і не виходив за межі дозволених сценаріїв.

3. `systemPrompt` фіксує, що результат пошуку повертає короткі відомості про листи разом з ідентифікаторами, а подальше читання або переміщення в Trash виконується через цей ідентифікатор, щоб агент послідовно переходив від знаходження листа до цільової дії.

4. `systemPrompt` зобов’язує викликати лише один інструмент за раз і чекати на його результат перед наступним кроком, щоб дії були послідовними та спиралися на підтверджений стан.

5. `systemPrompt` трактує переміщення в Trash як руйнівну дію, що зупиняється для людського підтвердження, тому агент не має вважати таку дію завершеною без явного результату.

6. `systemPrompt` вимагає, щоб за неоднозначного запиту агент спочатку ставив уточнювальне запитання і не викликав інструменти, щоб уникати помилкових дій над не тим листом.

7. `systemPrompt` визначає завершення сценарію коротким текстовим підсумком без виклику інструментів, щоб взаємодія закінчувалася зрозумілим для користувача результатом.

8. `systemPrompt` є read-only щодо зовнішніх сховищ і сам по собі не пише у ФС чи БД, а формує лише інструкцію поведінки агента.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).

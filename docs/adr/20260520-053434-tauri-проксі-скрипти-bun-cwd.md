# Tauri Проксі-скрипти у Кореневому package.json через bun --cwd

**Status:** Accepted
**Date:** 2026-05-20

## Context and Problem Statement

Проєкт `mlmail` має monorepo-структуру: кореневий `package.json` і workspace `app/`. Команди `tauri dev` і `tauri build` були доступні лише з директорії `app/`, що вимагало ручного `cd` перед кожним запуском.

## Considered Options

- `cd app && bun run tauri <cmd>` — зміна директорії через shell (відхилено)
- `bun --cwd=app run tauri <cmd>` — прапорець `bun` без зміни CWD оболонки

## Decision Outcome

Chosen option: "`bun --cwd=app run tauri <cmd>`", because флаг `--cwd` змінює лише внутрішній CWD процесу bun, не зачіпаючи середовище оболонки; підтверджено тестом `bun --cwd=app run tauri --version` → `tauri-cli 2.11.1`.

Додані скрипти у кореневий `package.json`:

```json
"dev":   "bun --cwd=app run tauri dev",
"tauri": "bun --cwd=app run tauri"
```

### Consequences

- Good, because `bun run dev` і `bun run tauri <args>` коректно резолвяться з кореня без зміни CWD оболонки.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

Змінений файл: кореневий `package.json`, скрипти `dev` і `tauri`. Верифіковано командою `bun run tauri --version` → `tauri-cli 2.11.1`. Bun версії `1.3.14`; підтримує `--cwd=<val>` з відносним шляхом.

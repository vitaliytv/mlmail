Ось практичний підхід — і одразу ваші реальні прогалини.

## 1. `bun test --coverage` (JS / Vue)

Я щойно прогнав — звіт вище. Колонки:

- **% Funcs / % Lines** — скільки функцій/рядків файлу виконалось під час тестів
- **Uncovered Line #s** — конкретні номери рядків, які тести не зачепили (у вас порожньо — усе, що завантажено, покрито)

## 2. Головна пастка звіту

`--coverage` показує **лише файли, які завантажились під час тестів**. Файл, на який немає **жодного** тесту, у звіт просто не потрапляє — і 100% виглядає оманливо.

Порівняння `find src` зі звітом дає справжню картину:

| Файл                                                                                                                                 | У звіті?     | Стан                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------ | ------------ | ------------------------------------------------------ |
| [auth-errors.js](app/src/i18n/auth-errors.js), [auth-store.js](app/src/services/auth-store.js), [Login.vue](app/src/views/Login.vue) | ✅           | покрито                                                |
| [App.vue](app/src/App.vue)                                                                                                           | ❌ відсутній | **0 тестів**                                           |
| [main.js](app/src/main.js)                                                                                                           | ❌ відсутній | **0 тестів** (bootstrap — зазвичай тестують через E2E) |

Тобто правило: **`coverage` файл не показав ≠ 100%, а можливо «взагалі не торкались»**. Звіряйте список файлів зі звіту з `find src`.

## 3. Rust (`src-tauri`, 20 файлів)

`cargo test` (ваш `test:rust`) проганяє тести, але **покриття не рахує**. Для цього окремий інструмент:

```bash
cargo llvm-cov --manifest-path src-tauri/Cargo.toml
```

(`cargo install cargo-llvm-cov`, якщо ще немає — дасть таблицю по `auth/*`, `gmail/*` тощо.)

## Що можу зробити далі

- додати скрипт `test:coverage` у [app/package.json](app/package.json) (+ за бажанням поріг `coverageThreshold` у конфізі, щоб CI падав при просіданні);
- написати тест на [App.vue](app/src/App.vue), щоб закрити прогалину.

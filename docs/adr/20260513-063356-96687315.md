## ADR Архітектура Google OAuth авторизації у MLMaiL

**Контекст:** MLMaiL — Tauri 2 + Vue 3 застосунок (macOS + Android), якому потрібен Google OAuth 2.0 з доступом до `gmail.modify`. Раніше жодного auth-шару не було; стартовий шаблон містив лише демо-команду `greet`.

**Рішення/Процедура/Факт:**

_Rust бекенд (`app/src-tauri/src/auth/`):_

- macOS flow: loopback TCP server (`tokio::net::TcpListener`), браузер через `tauri-plugin-opener`, PKCE (SHA-256 challenge у `pkce.rs`), обмін кодом у `token_exchange.rs` з `FlowKind::Desktop`
- Android flow: Tauri 2 mobile plugin pattern (`@TauriPlugin` у `MlmailAuthPlugin.kt`) — Rust викликає Kotlin через `PluginHandle`; Kotlin uses `CredentialManager` для sign-in + `AuthorizationClient` для server auth code; `FlowKind::Android` — PKCE не використовується, redirect_uri пустий
- Google може ротувати refresh_token у відповіді на refresh — завжди зберігати новий при наявності
- Rust — єдиний тримач токенів; Vue отримує access token тільки через Tauri команду `auth_get_access_token`
- Keychain: `keyring` crate + `apple-native` feature на macOS; `EncryptedSharedPreferences` (AES256_GCM) у Kotlin на Android

_Vue фронтенд (`app/src/`):_

- `Login.vue` — весь UI українською; рядки кнопок/статусів hardcoded у `.vue`
- `auth-store.js` — singleton composable; `errorKind` зберігає Rust enum-рядок
- `auth-errors.js` — маппінг `kind` ("Cancelled", "Network", "OAuth"…) → українська строка; Rust повертає тільки код, не локалізований рядок

_Google Cloud Setup:_

- OAuth consent screen Internal type (для `@nitralabs.com`) створено через `gcloud iap oauth-brands create` — це дає `orgInternalOnly: true` і скасовує вимогу Google App Verification для `gmail.modify`
- Три OAuth clients: Desktop application (macOS loopback), Android (`com.vitaliytv.mlmail` + SHA-1), Web application (serverClientId для Credential Manager — redirect URIs порожні)
- Client IDs зберігаються у `app/src-tauri/.env` (gitignored), компілюються через `option_env!()` у `config.rs`

_Тести (TDD):_ 32 Rust unit-тестів (`pkce`, `id_token`, `state`, `token_exchange` з mockito, `in_memory` storage) + 22 JS-тестів (vitest + `@vue/test-utils`)

**Обґрунтування:** Loopback flow — єдиний безпечний OAuth варіант для нативних macOS-застосунків без серверного компонента. Credential Manager — актуальний (не deprecated) Google-рекомендований шлях для Android. Таuri 2 mobile plugin pattern обраний замість сирого JNI — офіційно підтримуваний і типобезпечний. Rust як єдиний тримач токенів — токени не потрапляють у Vue-контекст/WebView. Internal org type (nitralabs.com) усуває 30-денний Google verification процес.

**Розглянуті альтернативи:** AppAuth-Android (окрема бібліотека, зайня залежність), `google_sign_in` deprecated SDK, сировий JNI замість Tauri plugin (відкинуто — складніше і менш maintainable), `keyring` зі Stronghold backend (зайвий рівень абстракції), зберігання токенів у Vue/localStorage (відкинуто — безпека).

**Зачіпає:** `app/src-tauri/src/auth/` (13 файлів), `app/src-tauri/src/lib.rs`, `app/src-tauri/Cargo.toml`, `app/src-tauri/gen/android/.../auth/` (4 .kt файли), `app/src-tauri/gen/android/app/build.gradle.kts`, `app/src/views/Login.vue`, `app/src/services/auth-store.js`, `app/src/i18n/auth-errors.js`, `app/src/App.vue`, `docs/adr/ADR-0006-google-oauth.md`, `docs/ci4/02-containers.md`, `docs/ci4/03-components.md`, `docs/ci4/04-code.md`, `docs/ci4/decisions.md`

## Knowledge IAP OAuth Admin API deprecated — brand creation через gcloud недоступна після Mar 2026

**Контекст:** Для MLMaiL OAuth consent screen Internal type (тільки `@nitralabs.com`) brand було створено через `gcloud iap oauth-brands create`. Google повідомляє про закриття IAP OAuth Admin APIs.

**Рішення/Процедура/Факт:** `gcloud iap oauth-brands create/list` видає WARNING: "permanently shut down March 19, 2026." Після цієї дати brand потрібно буде редагувати тільки через Google Auth Platform UI (`https://console.cloud.google.com/auth/`). OAuth clients (Desktop/Android/Web) ніколи не підтримувались через gcloud — тільки UI.

**Обґрунтування:** Команда `gcloud iap oauth-brands create` відноситься до IAP (Identity-Aware Proxy) OAuth Admin API, яке Google закриває. Самі brands і клієнти залишаться робочими — тільки CLI/API для їх управління буде відключено.

**Розглянуті альтернативи:** Не обговорювались — відкриття цього факту вийшло з термінального виводу.

**Зачіпає:** Процедура початкового налаштування Google Cloud для нових розробників MLMaiL; `docs/adr/ADR-0006-google-oauth.md` розділ про Google Cloud Setup.

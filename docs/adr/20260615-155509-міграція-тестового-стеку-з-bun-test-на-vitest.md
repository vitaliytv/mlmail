## ADR Вбудований Google OAuth Desktop client_secret як вбудований fallback

## Context and Problem Statement
Desktop-клієнт Google OAuth потребує `client_secret` при обміні коду на токен. Тримання секрету у `.env.secret` (gitignored) вимагає ручного налаштування для кожного середовища. Водночас Google прямо вказує, що `client_secret` для типу **Desktop app** не є конфіденційним — він усе одно потрапляє в бінарник і будь-хто може витягти його з застосунку.

## Considered Options
* Вшити `client_secret` у вихідний код як вбудований fallback (`.env.secret` має пріоритет якщо заданий)
* Тримати виключно в `.env.secret` (gitignored), ніколи не вшивати в код

## Decision Outcome
Chosen option: "Вшити `client_secret` у вихідний код як вбудований fallback", because для Desktop-типу OAuth секрет і так потрапляє в бінарник при будь-якому підході; вшивання спрощує onboarding без втрати захищеності (PKCE захищає флоу, а не сам секрет); користувач явно авторизував це рішення.

### Consequences
* Good, because transcript фіксує очікувану користь: `desktop_client_secret()` бере env/`.env.secret` якщо задано, інакше повертає вбудований рядок — налаштування `.env.secret` стає необов'язковим для desktop.
* Bad, because секрет потрапив у git-історію та в цей чат — якщо репо стане публічним або історія витече, ротація обов'язкова; transcript це фіксує.
* Bad, because trufflehog без виключення заблокував би lint-security — додано `app/src-tauri/src/auth/config\.rs$` у `.trufflehog-exclude`.

## More Information
Змінені файли: `app/src-tauri/src/auth/config.rs` (`const DESKTOP_SECRET_BUILTIN: &str = "GOCSPX-…"; // cspell:disable-line`; функція `desktop_client_secret()` з fallback-логікою), `.trufflehog-exclude` (додано рядок-виняток).

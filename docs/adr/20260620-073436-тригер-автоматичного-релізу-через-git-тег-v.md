## Context and Problem Statement

## Considered Options
* Залишити `Swatinem/rust-cache` (можливо з `zizmor:ignore` коментом)
* Видалити `Swatinem/rust-cache` — релізи рідкісні, кеш некритичний

## Decision Outcome
Chosen option: "Видалити `Swatinem/rust-cache`", because релізи відбуваються рідко і втрата кешу Rust у них не критична, тоді як cache-poisoning у release-контексті — реальний ризик, підтверджений zizmor.

* Bad, because `build-android` на Ubuntu збирає Rust-залежності з нуля при кожному релізі, що збільшує час збірки.

## More Information
Крок прибрано з обох jobs: `build-desktop` і `build-android`. Перевірка: `bun run lint-ga 2>&1 | grep "release.yml"` → `🌈 completed ./.github/workflows/release.yml`.

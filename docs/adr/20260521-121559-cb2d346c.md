## `webapp-testing` — детально

**Джерело:** `github.com/anthropics/skills` (`skills/webapp-testing`)

### Що робить

Управляє браузером через **Playwright** для тестування локального веб-застосунку. Конкретно:
- відкриває сторінку за URL (`http://localhost:PORT`)
- робить **скріншоти** — фіксує поточний стан UI
- зчитує **console logs** (JS-помилки, попередження)
- досліджує DOM — знаходить елементи, перевіряє їхній стан
- клікає/заповнює форми/взаємодіє з UI і перевіряє результат

### Тригер (дослівно з SKILL.md)
> "Use this skill when the user asks to test, verify, or debug a local web application."

Тобто активується, коли ти кажеш щось на кшталт «перевір, що логін працює» або «зроби скріншот сторінки після змін».

### Інструменти (helper scripts)
| Скрипт | Дія |
|--------|-----|
| `console_logging.py` | Збирає JS console.log / warnings / errors у браузері |
| `element_discovery.py` | Знаходить елементи на сторінці по селектору / тексту |
| + ще кілька | Взаємодія (клік, ввід) і скріншоти |

### Залежності
- **Playwright** (Python-бібліотека, `pip install playwright`)
- **запущений dev-сервер** (Playwright лише відкриває URL — застосунок має вже крутитись)

### Як підходить до твого проєкту

Твій застосунок — **Vite + Vue3 + Tauri**. Vite dev-сервер (`bun run dev` у `app/`) слухає стандартний порт (5173). Саме туди `webapp-testing` і відкриватиме браузер.

**Підходить для:**
- Перевірити, що компонент відрендерився після зміни коду
- Зловити JS-помилки в консолі (яких `bun test` не бачить)
- E2E-сценарії: авторизація, навігація, відправка форм

**Не замінює** юніт-тести у `scripts/__tests__/` — вони тестують бізнес-логіку без браузера і не потребують запущеного сервера.

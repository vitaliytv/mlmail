Infisical має 4 типи проєктів, які відрізняються **призначенням** і доступним інструментарієм:

## SecretOps (стандартний вибір)
Зберігання секретів у форматі **key-value** по середовищах (`dev`, `staging`, `prod`).
- Підходить для env-vars, API-ключів, паролів, base64-blob.
- Саме цей тип ми описували для Android keystore — кладемо `ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD` тощо.
- Має SDK, CLI, GitHub-синк, Docker-інтеграції.

## PKI (Public Key Infrastructure)
Повноцінний **Certificate Authority** всередині Infisical.
- Виписує X.509 сертифікати (SSL/TLS, mutual TLS).
- Керує ланцюжками CA, часом дії сертів, списками відкликання (CRL).
- Не для звичайних секретів — суто для PKI-операцій.

## SSH
Управління **SSH-сертифікатами** (не ключами, а саме сертами).
- Infisical виступає як SSH CA: підписує тимчасові сертифікати для доступу до серверів.
- Дозволяє відмовитись від `authorized_keys` на кожному хості — замість цього всі хости довіряють CA.
- Для Android-флоу не потрібне.

## KMS (Key Management Service)
**Шифрування даних** через API Infisical.
- Зберігаєш не сам секрет, а **CMK (Customer Master Key)**.
- Infisical шифрує/дешифрує дані за запитом (Envelope Encryption).
- Аналог AWS KMS / Google Cloud KMS, але self-hostable.
- Підходить для шифрування колонок в БД, файлів тощо.

## Для вас (Android keystore)
**SecretOps** — і більше нічого.

```
Project Type: SecretOps
Environment: prod
Secrets: ANDROID_KEYSTORE_BASE64, ANDROID_KEYSTORE_PASSWORD,
ANDROID_KEY_ALIAS, ANDROID_KEY_PASSWORD
```

PKI/KMS/SSH — це окремі продукти для інших задач; їх варто знати, щоб не плутатись при створенні проєкту в UI.

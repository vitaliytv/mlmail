Я не можу запустити `tauri android init` через обмеження на ENV-змінні в одній команді. Зроби це вручну:

```bash
cd /Users/vitalii/www/nitra/task/app
export ANDROID_HOME=/Users/vitalii/Library/Android/sdk
export NDK_HOME=/Users/vitalii/Library/Android/sdk/ndk/30.0.14904198
bun run tauri android init
```

Після цього поверніться — я:
1. Прочитаю згенерований `build.gradle.kts`
2. Додам release-підпис (як у mlmail/myshare)
3. Оновлю `release.yml` — додам `build-android` job
4. Згенерую keystore для task

Поки що для task є лише DMG. Keystore потрібен буде після ініціалізації Android.

## ADR

```markdown
## ADR Автоматичний реліз Tauri-застосунків через GitHub Actions з підписом через Infisical

## Context and Problem Statement
Три Tauri 2 монорепо (mlmail, myshare, task) потребують автоматичного публікування DMG (macOS) і APK (Android) при кожному git-тегу. Keystore-секрети для підпису APK мають безпечно зберігатися поза репозиторіями та синхронізуватися в GitHub Actions Secrets.

## Considered Options
* Тригер на push git-тегу `v*` → GitHub Release + artifact upload через `gh release upload`
* Зберігання keystore-секретів у Infisical (Варіант А — автосинк в GitHub Actions Secrets)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Push git-тегу `v*` → `.github/workflows/release.yml` з джобами `create-release`, `build-desktop`, `build-android`; секрети через Infisical → GitHub Actions Secrets", because це описано в `docs/android-release-keystore.md` як «Варіант А» і дозволяє централізовано керувати keystore без прямого доступу до Infisical CLI у CI.

### Consequences
* Good, because секрети не зберігаються в репозиторії; `keystore.properties` у `.gitignore`; локальний `tauri android dev/build` працює через fallback на debug-підпис у `build.gradle.kts`.
* Bad, because task потребує окремого кроку `tauri android init` перед увімкненням APK-збірки; `infisical login` локально не працює, тому секрети вносяться через дашборд вручну.

## More Information
- Файли: `.github/workflows/release.yml` (mlmail, myshare, task); `app/src-tauri/gen/android/app/build.gradle.kts` (mlmail, myshare)
- Keystore: `~/keystores-release/{mlmail,myshare}/` — поза git, секрети у `secrets/*.txt`
- `zizmor.yml` — `unpinned-uses: ref-pin`; `rust-cache` прибраний через cache-poisoning у release-контексті
- Infisical → GitHub інтеграція: Project → Integrations → GitHub → env `prod` → GitHub Actions Secrets
- task: `bun run tauri android init` (потрібен `ANDROID_HOME`, `NDK_HOME`) для активації APK-збірки
```

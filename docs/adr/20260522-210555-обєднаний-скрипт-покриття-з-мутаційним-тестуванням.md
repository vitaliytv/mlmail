## ADR: Об'єднаний скрипт покриття з мутаційним тестуванням

## Context and Problem Statement

Проєкт мав окремі команди для покриття коду (JS + Rust) і не мав мутаційного тестування. Після додавання `cargo-llvm-cov` і StrykerJS постала потреба звести всі метрики в один git-trackable файл, доступний для порівняння між запусками.

## Considered Options

- Один скрипт `scripts/coverage.js` з єдиним прогоном і єдиною таблицею в `COVERAGE.md`
- Два окремих скрипти (`coverage.js` + `mutation.js`) з двома секціями у `COVERAGE.md`
- Склейка команд через `&&` у `package.json` без скрипта-оркестратора

## Decision Outcome

Chosen option: "Один скрипт `scripts/coverage.js`", because користувач явно вказав «один скрипт, один запуск»; дві секції у файлі ускладнюють синхронізацію без додаткової вартості.

### Consequences

- Good, because `bun run coverage` — єдина точка входу; `COVERAGE.md` без таймстампу даєчистий git-diff лише при реальній зміні метрик.
- Bad, because повний прогін займає ~12–15 хв (мутації Rust + JS послідовно); паралельний запуск двох сесій ламає `inPlace`-стан Stryker — закрито lock-механізмом.

## More Information

- `scripts/coverage.js` — оркестратор; lock-файл `.coverage.lock` + PID-перевірка живучості
- `app/stryker.config.mjs` — `testRunner: command`, `inPlace: true`, `mutate: src/**/*.{js,vue}`
- `.gitignore`: `mutants.out/`, `app/reports/`, `.coverage.lock`
- `COVERAGE.md` — таблиця: Область | Рядки | Функції | Вбито мутацій | Score

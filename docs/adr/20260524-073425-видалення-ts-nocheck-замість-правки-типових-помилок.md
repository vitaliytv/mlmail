## ADR Видалення `// @ts-nocheck` замість правки типових помилок

## Context and Problem Statement
Попередній агент заглушував TypeScript-помилки у `.js`-файлах проєкту (тестових файлах, `vite.config.js`, `happy-dom.preload.js`, `stryker.config.mjs`, `vite-env.d.ts`) додаванням директиви `// @ts-nocheck` на першому рядку. Жодне правило в `.cursor/rules/` цього не вимагало. Стало питання — видалити заглушки чи лишити.

## Considered Options
* Видалити `// @ts-nocheck` і перевірити, чи `tsc` видає реальні помилки
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "Видалити `// @ts-nocheck` і перевірити `tsc`", because `bunx tsc -p app/jsconfig.json --noEmit` (TS 6.0.3) після видалення директив повернув exit 0 — усі файли вже типобезпечні зі `strict: true`.

### Consequences
* Good, because `vite-env.d.ts` повернувся до канонічного стану (один рядок `/// <reference types="vite/client" />`), що відповідає правилу `n-vue.mdc`.
* Good, because transcript фіксує очікувану користь: tsc виявив 0 помилок — заглушки були виключно карго-культом.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінені файли: `app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/src/vite-env.d.ts`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/stryker.config.mjs`. Файл `app/auto-imports.d.ts` пропущено — він генерується `unplugin-auto-import` і `// @ts-nocheck` там проставляє сам плагін. `app/jsconfig.json` має `"include": ["src/**/*"]` — файли поза `src/` (`vite.config.js`, `stryker.config.mjs`, `happy-dom.preload.js`) взагалі не охоплюються type-check'ом. Перевірка: `bunx tsc -p app/jsconfig.json --noEmit` → exit 0; окрема перевірка out-of-src файлів із тими самими strict-опціями → exit 0.

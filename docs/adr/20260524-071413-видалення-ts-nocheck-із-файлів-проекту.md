## ADR Видалення `// @ts-nocheck` із файлів проекту

## Context and Problem Statement
Попередній агент додав `// @ts-nocheck` на початок дев'яти файлів (`app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/src/vite-env.d.ts`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/stryker.config.mjs`) замість того, щоб вирішити реальні причини TS-помилок. Це суперечило правилу `n-vue.mdc`, яке вимагає, щоб `vite-env.d.ts` містив рівно один рядок.

## Considered Options
* Залишити `// @ts-nocheck` і не чіпати TS-конфіг
* Видалити `// @ts-nocheck` і перевірити, чи є реальні TS-помилки (`tsc --noEmit`)

## Decision Outcome
Chosen option: "Видалити `// @ts-nocheck` і перевірити, чи є реальні TS-помилки", because `bunx tsc -p app/jsconfig.json --noEmit` (TypeScript 6.0.3) повернув exit 0 без жодної помилки — директиви були повністю зайві.

### Consequences
* Good, because `app/src/vite-env.d.ts` знову відповідає канону `n-vue.mdc` (один рядок `/// <reference types="vite/client" />`).
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- `app/jsconfig.json` використовує `"strict": true`, `"allowJs": true`, `include: ["src/**/*"]` — файли поза `src/` взагалі не type-чекаються цим конфігом.
- `app/auto-imports.d.ts` збережено з `// @ts-nocheck`, бо його генерує `unplugin-auto-import` автоматично.
- Команда верифікації: `bunx tsc -p app/jsconfig.json --noEmit` (exit 0).
- Додатково: `.gitignore` розширено з `mutants.out/` до `mutants.out*/`, щоб покрити `mutants.out.old/`, який створює `cargo-mutants` при повторних прогонах.

## ADR Розширення `.gitignore` для артефактів `cargo-mutants`

## Context and Problem Statement
`cargo-mutants` при кожному повторному запуску перейменовує поточну директорію `mutants.out/` у `mutants.out.old/` перед початком нового прогону. У `.gitignore` був лише патерн `mutants.out/`, тому `mutants.out.old/` (з файлами `caught.txt`, `debug.log`, `missed.txt`, `mutants.json` тощо) потрапляла в `git status` як untracked.

## Considered Options
* Додати окремий рядок `mutants.out.old/` до `.gitignore`
* Замінити `mutants.out/` на glob `mutants.out*/`

## Decision Outcome
Chosen option: "Замінити `mutants.out/` на glob `mutants.out*/`", because один glob покриває всі можливі варіанти ротації (`.old`, потенційні `.1`, `.bak` тощо) без дублювання рядків.

### Consequences
* Good, because `git check-ignore -v app/src-tauri/mutants.out.old/caught.txt` підтвердив, що файл тепер ігнорується (`.gitignore:39:mutants.out*/`).
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Змінено рядок у `.gitignore`: `mutants.out/` → `mutants.out*/`
- Артефакти знаходяться в `app/src-tauri/mutants.out.old/`: `caught.txt`, `debug.log`, `diff/`, `lock.json`, `log/`, `missed.txt`, `mutants.json`, `outcomes.json`, `timeout.txt`, `unviable.txt`.

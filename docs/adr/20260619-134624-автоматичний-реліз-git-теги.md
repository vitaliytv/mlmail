# Автоматичний реліз Tauri-застосунків через git-теги

**Status:** Accepted
**Date:** 2026-06-19

## Context and Problem Statement

Три Tauri-монорепо (mlmail, myshare, task) потребували автоматичної публікації DMG і APK на GitHub Releases без ручних кроків. Раніше жодного release-workflow не існувало. Вимога: zizmor-лінт без findings і відсутність сторонніх upload-екшенів із потенційним cache-poisoning.

## Considered Options

* Push git-тегу `v*` як тригер + `actions/github-script` для створення Release + `gh release upload` для артефактів
* `softprops/action-gh-release@v2` як upload-екшен (відкинуто: zizmor cache-poisoning-попередження)
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

Chosen option: "push git-тегу `v*` + `gh release upload`", because `gh` CLI присутній у всіх GitHub-hosted runners без `ref-pin`; `softprops/action-gh-release` спричиняв zizmor findings, тоді як `gh release upload` забезпечує `bun run lint-ga` → `No findings to report (35 suppressed)`.

### Consequences

* Good, because `git tag vX.Y.Z && git push origin vX.Y.Z` автоматично публікує `.dmg` і `.apk` без ручних кроків.
* Good, because без `Swatinem/rust-cache@v2` у release-workflow zizmor проходить чисто — прийнятно, релізи рідкісні.
* Bad, because без `rust-cache` кожен release-білд компілює Rust з нуля.
* Bad, because `nitra/task` отримав лише `build-desktop` — `gen/android` не ініціалізований, APK потребує `tauri android init`.

## More Information

Файли: `.github/workflows/release.yml` (mlmail, myshare, task). Джоби: `create-release` (`actions/github-script`) → паралельно `build-desktop` (macOS universal DMG) і `build-android` (Ubuntu APK). Android підпис у `app/src-tauri/gen/android/app/build.gradle.kts`: `signingConfigs { create("release") { … } }` + `signingConfig = if (keystorePropertiesFile.exists()) signingConfigs.getByName("release") else signingConfigs.getByName("debug")` (fallback для локальної розробки без `keystore.properties`). Секрети (mlmail, myshare): через Infisical → GitHub Actions Secrets. zizmor-конфіг: `.github/zizmor.yml` (`ref-pin`, `template-injection: disable`). Завантаження: `gh release upload "${{ github.ref_name }}" <glob>`. Наступний крок для task APK: `tauri android init`, коміт `gen/android`, правка `build.gradle.kts`, додавання `build-android`-джоби.

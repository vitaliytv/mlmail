## Knowledge Tauri e2e: macOS не підтримує WebKit WebDriver

**Контекст:** Питання про підходи до e2e-тестування Tauri-застосунку, зокрема на macOS.

**Рішення/Процедура/Факт:** Офіційний e2e-стек — `tauri-driver` + WebDriverIO/Selenium. На **macOS `tauri-driver` офіційно не підтримується** — Apple не надає публічний WebKit WebDriver. Реальні e2e з Tauri runtime гоняють тільки в Linux CI (`ubuntu-latest` + `xvfb-run` + `webkit2gtk-driver`). На mac-девелоперських машинах e2e покривають двома способами: (1) юніт-тести Rust для `#[tauri::command]`; (2) Vitest/Playwright проти dev-сервера з мок-замінниками `@tauri-apps/api` (без Tauri runtime).

**Обґрунтування:** Обмеження платформенне — Apple не публікує WebKit WebDriver API. Обхід через Linux CI є стандартною практикою в екосистемі Tauri.

**Розглянуті альтернативи:** Не обговорювалися; обмеження не має обходу на рівні самого `tauri-driver`.

**Зачіпає:** CI-конфігурація (`.github/workflows/`), тестова інфраструктура (`wdio.conf.js`, `tauri-driver`), будь-які майбутні e2e-сьюти для Tauri-частини проєкту.

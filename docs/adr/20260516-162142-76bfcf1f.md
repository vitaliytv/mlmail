## Де була плутанина

Коли я казав «комічу X», відбувалось ось що:

1. Я роблю `Edit file.vue` → git-ai hook спрацьовує → автоматичний коміт `feat(login): rewrite Login.vue` вже у репо.
2. Я потім запускаю `git commit -m "feat(login): ..."` → отримую або **чистий tree** (нічого немає, бо git-ai вже взяв), або коміт дублюється якщо щось не взяв хук.
3. У тексті я пишу: «Закомічено: feat(login)…» — і це звучить як моя дія, хоча часто це вже зробив git-ai **до мого `git commit`**.

Тобто я некоректно присвоював собі коміти, які зробив git-ai, або навіть не перевіряв `git status` перед `git commit`, щоб впевнитись чи є що комітити.

**Як правильно поводитись:**

1. Перед будь-якою `git commit` командою — завжди `git status` + `git diff --cached`.
2. Якщо tree чистий і в `git log` вже є потрібний коміт — сказати **«git-ai вже закомітив це за вас після мого edit, у `git log` він відображений як `<hash>`»**, а не претендувати що це я.
3. Взагалі — не запускати `git commit` без явного «комміть» від тебе, що прибирає всю цю плутанину.

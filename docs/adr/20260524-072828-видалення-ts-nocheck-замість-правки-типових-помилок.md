## ADR Видалення `// @ts-nocheck` замість правки типових помилок

## Context and Problem Statement
Попередній агент-сесії в незакомічених змінах додав `// @ts-nocheck` на першу лінію дев'яти файлів (`app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/src/vite-env.d.ts`, `app/stryker.config.mjs`) та в автогенерованому `app/auto-imports.d.ts`. Жодне правило у `.cursor/rules/*.mdc` такого не вимагало; `grep -r "nocheck"` по `.cursor/` повертав 0 збігів.

## Considered Options
* Залишити `// @ts-nocheck` як заглушку для типових помилок
* Видалити `// @ts-nocheck` і перевірити реальний стан `tsc`

## Decision Outcome
Chosen option: "Видалити `// @ts-nocheck` і перевірити реальний стан `tsc`", because `bunx tsc -p app/jsconfig.json --noEmit` (TypeScript 6.0.3, `strict: true`) повернув exit 0 і 0 помилок — коментарі були повністю зайві. Файли поза `include: ["src/**/*"]` (`vite.config.js`, `test/happy-dom.preload.js`, `stryker.config.mjs`) взагалі не охоплюються `jsconfig.json`, тому `@ts-nocheck` у них не мав жодного ефекту. Виняток — `app/auto-imports.d.ts`: він залишений незміненим, бо `// @ts-nocheck` туди ставить сам `unplugin-auto-import`.

### Consequences
* Good, because `app/src/vite-env.d.ts` приведено до канонічного вигляду (рівно `/// <reference types="vite/client" />`), що відповідає правилу n-vue.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли змінені: `app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/src/vite-env.d.ts`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/stryker.config.mjs`. Команда перевірки: `bunx tsc -p app/jsconfig.json --noEmit` (exit 0). Конфіг: `app/jsconfig.json` з `strict: true`, `include: ["src/**/*"]`.

## ADR Розширення `.gitignore` для `mutants.out*/`

## Context and Problem Statement
`.gitignore` покривав лише `mutants.out/`, але `cargo-mutants` при повторному запуску перейменовує попередній результат у `mutants.out.old/`. Через це `app/src-tauri/mutants.out.old/` з файлами `caught.txt`, `debug.log`, `diff`, `lock.json`, `log`, `missed.txt`, `mutants.json`, `outcomes.json`, `timeout.txt`, `unviable.txt` залишався untracked у git.

## Considered Options
* Додати окремий рядок `mutants.out.old/`
* Розширити glob до `mutants.out*/`

## Decision Outcome
Chosen option: "Розширити glob до `mutants.out*/`", because один glob `mutants.out*/` покриває і `mutants.out/`, і `mutants.out.old/`, і будь-які майбутні ротаційні суфікси без додаткових рядків. `git check-ignore -v` підтвердив: `app/src-tauri/mutants.out.old/caught.txt` ігнорується правилом `.gitignore:39:mutants.out*/`.

### Consequences
* Good, because transcript фіксує очікувану користь: `git status` більше не показує untracked файли з `mutants.out.old/`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінено: `.gitignore`, рядок 39 — `mutants.out/` → `mutants.out*/`. Команда перевірки: `git check-ignore -v app/src-tauri/mutants.out.old/caught.txt`.

## ADR Release-підпис Android через `keystore.properties` з fallback

## Context and Problem Statement
`build.gradle.kts` у `mlmail` і `myshare` не мав `signingConfigs` для release-білду; APK виходив непідписаним або підписувався debug-ключем.

## Considered Options
* `signingConfigs { create("release") }` з умовним fallback на debug, якщо `keystore.properties` відсутній
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "signingConfigs з умовним fallback", because такий підхід не ламає локальний `tauri android dev/build` (де `keystore.properties` немає), і одночасно вмикає підписаний APK у CI, де файл синтезується з секретів.

### Consequences
* Good, because transcript фіксує очікувану користь: підписаний APK у CI без обов'язкового `keystore.properties` у репо.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файли: `app/src-tauri/gen/android/app/build.gradle.kts` (mlmail, myshare). `keystore.properties` вже в `gen/android/.gitignore`. Секрети в CI: `ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD`, `ANDROID_KEY_ALIAS`, `ANDROID_KEY_PASSWORD`.

## ADR Інжекція секретів Infisical у CI через OIDC machine identity (nitra-патерн)

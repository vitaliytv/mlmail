* Інші варіанти в transcript не обговорювалися.

## Decision Outcome

## Considered Options
* `softprops/action-gh-release@v2` для завантаження артефактів

## Decision Outcome

### Consequences

## ADR Android підпис через keystore.properties із fallback на debug-підпис

## Context and Problem Statement
`tauri android build` (release-варіант) потребує підписаного APK. Секрети keystore мають надходити з CI (Infisical → GitHub Secrets) і не повинні ламати локальну розробку, де `keystore.properties` відсутній.

## Considered Options
* `signingConfigs.release` з `keystorePropertiesFile.exists()`-перевіркою + fallback на `signingConfigs.debug`
* Інші варіанти в transcript не обговорювалися.

## Decision Outcome
Chosen option: "умовний `signingConfig` з fallback на debug", because це дозволяє `tauri android dev` / `tauri android build` запускатися локально без keystore, а CI використовує `keystorePropertiesFile` з секретами `ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD`, `ANDROID_KEY_ALIAS`, `ANDROID_KEY_PASSWORD`.

### Consequences
* Good, because transcript фіксує очікувану користь: локальний білд не зламається за відсутності `keystore.properties`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Змінені файли: `app/src-tauri/gen/android/app/build.gradle.kts` у mlmail і myshare. Блоки `signingConfigs { create("release") { … } }` та `signingConfig = if (keystorePropertiesFile.exists()) signingConfigs.getByName("release") else signingConfigs.getByName("debug")`. Секрети керуються через Infisical Варіант А (sync до GitHub Actions Secrets).

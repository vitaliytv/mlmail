# Видалення `// @ts-nocheck` із JS-файлів проекту

**Status:** Accepted
**Date:** 2026-05-24

## Context and Problem Statement

У робочому дереві (uncommitted) виявлено `// @ts-nocheck` у перших рядках дев'яти файлів: тестові `.test.js`, `vite.config.js`, `stryker.config.mjs`, `happy-dom.preload.js`, `test-utils/quasar.js` та `vite-env.d.ts`. Жодне правило у `.cursor/rules/*.mdc` цього не вимагало (`grep -r "nocheck" .cursor/` → 0 збігів). Директива у `vite-env.d.ts` порушувала правило `n-vue.mdc`, яке вимагає рівно один рядок `/// <reference types="vite/client" />`.

## Considered Options

- Залишити `// @ts-nocheck` — заглушувати потенційні TS-помилки без з'ясування причини.
- Видалити `// @ts-nocheck` і перевірити `tsc --noEmit` — з'ясувати, чи виникають реальні помилки.

## Decision Outcome

Chosen option: "Видалити `// @ts-nocheck` і перевірити `tsc --noEmit`", because `bunx tsc -p app/jsconfig.json --noEmit` (TypeScript 6.0.3, `strict: true`) завершився з exit 0 і нульовою кількістю помилок — директиви були повністю зайві. Файли поза `include: ["src/**/*"]` (`vite.config.js`, `test/happy-dom.preload.js`, `stryker.config.mjs`) взагалі не охоплюються `jsconfig.json`, тому `@ts-nocheck` у них не мав жодного ефекту.

### Consequences

- Good, because `app/src/vite-env.d.ts` приведено до канонічного вигляду `/// <reference types="vite/client" />`, що відповідає правилу `n-vue.mdc`.
- Good, because кілька файлів після видалення зрівнялись із HEAD — `// @ts-nocheck` виявився їхньою єдиною незакомміченою зміною.
- Bad, because transcript не містить підтверджених негативних наслідків.

## More Information

- Файли змінено: `app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/src/vite-env.d.ts`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/stryker.config.mjs`.
- `app/auto-imports.d.ts` залишено без змін — `// @ts-nocheck` у ньому ставить сам `unplugin-auto-import`.
- `app/jsconfig.json`: `"strict": true`, `"allowJs": true`, `"include": ["src/**/*"]`.
- Команди перевірки: `bunx tsc -p app/jsconfig.json --noEmit` (exit 0); `bunx tsc --noEmit --allowJs --strict … vite.config.js test/happy-dom.preload.js stryker.config.mjs` (exit 0).

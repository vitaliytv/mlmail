## Inbox count: endpoint choice

**Контекст:** Стартовий екран MLMaiL показує кількість листів у мітці INBOX.

**Рішення/Процедура/Факт:** Точне `messagesTotal` із `GET users/me/labels/INBOX`. HTTPS-виклик — у Rust-модулі `app/src-tauri/src/gmail/`, через спільний `auth::acquire_access_token`. Tauri-команда — `gmail_inbox_count`. Vue Auth Store тримає число у `inboxCount` ref і скидає його у стан «не залогінений» при `ReauthRequired` від Gmail.

**Обґрунтування:** Один HTTP-запит, точне число (а не estimate), мінімальний scope (вже мається `gmail.modify`). HTTPS у Rust узгоджено з ADR-0006 (token surface).

**Розглянуті альтернативи:**

- `users.getProfile` — рахує всю поштову скриньку, не лише INBOX. Відкинуто.
- `messages.list?labelIds=INBOX` + `resultSizeEstimate` — приблизне, не точне. Відкинуто.

**Зачіпає:** Auth Store, Login.vue, новий Rust-модуль `gmail/`, i18n (`Http`/`Parse`).

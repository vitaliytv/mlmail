## Knowledge Tauri vs Capacitor: технічний огляд для команди

**Контекст:** Команда розглядає Tauri як альтернативу Capacitor для десктопних і мобільних застосунків; потрібно пояснити відмінності, переваги та обмеження для технічного колективу.

**Рішення/Процедура/Факт:**
Tauri використовує системний WebView (без вбудованого Chromium) і Rust-бекенд; Capacitor — Cordova-подібна обгортка над WebView з мостом на JS. На Android Tauri використовує Android NDK через Rust (без JVM/Java у критичному шляху), тоді як Capacitor завжди вимагає Java/Kotlin для плагінів.

**Обґрунтування:** Tauri дає значно менший розмір бінарника (немає вбудованого Chromium), прямий доступ до нативних API через Rust FFI, і можливість викликати ML-моделі локально (llama.cpp, ort тощо) без Bridge-оверхеду.

**Розглянуті альтернативи:** Capacitor (JS-міст + Java плагіни), Electron (вбудований Chromium, більший бінарник).

**Зачіпає:** вибір фреймворку для нових проєктів команди; Android-специфіка (NDK vs JVM); інтеграція локального AI.

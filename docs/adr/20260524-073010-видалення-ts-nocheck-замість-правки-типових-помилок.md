## ADR Видалення `// @ts-nocheck` замість правки типових помилок

## Context and Problem Statement
У робочому дереві репо `mlmail` попередній агент додав `// @ts-nocheck` у рядок 1 дев'яти файлів (`app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/src/vite-env.d.ts`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/stryker.config.mjs`) — очевидно, щоб заглушити помилки TypeScript. `app/jsconfig.json` має `"strict": true` і `"allowJs": true`, тому Volar/tsc type-чекає `.js`-файли включно з тестами та конфіг-файлами. Крім того, для `app/src/vite-env.d.ts` директива порушувала канон `n-vue.mdc`, який вимагає, щоб файл містив рівно один рядок `/// <reference types="vite/client" />`.

## Considered Options
* Прибрати `// @ts-nocheck` і перевірити реальні типові помилки через `tsc --noEmit`
* Залишити `// @ts-nocheck` як є

## Decision Outcome
Chosen option: "Прибрати `// @ts-nocheck` і перевірити реальні типові помилки через `tsc --noEmit`", because `// @ts-nocheck` приховував помилки замість того, щоб їх вирішити; перевірка показала, що `bunx tsc -p app/jsconfig.json --noEmit` (TypeScript 6.0.3) повертає **exit 0** без жодної помилки — директиви були повністю зайвими.

### Consequences
* Good, because transcript фіксує очікувану користь: `tsc` exit 0 після видалення `// @ts-nocheck` — реальних типових помилок не виявлено, код типобезпечний.
* Good, because `app/src/vite-env.d.ts` повернено до канонічного вигляду `/// <reference types="vite/client" />`, що відповідає `n-vue.mdc`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Файли поза `src/` (`vite.config.js`, `test/happy-dom.preload.js`, `stryker.config.mjs`) взагалі не охоплені `"include": ["src/**/*"]` у `app/jsconfig.json` — для них `// @ts-nocheck` нічого не давав.
- `app/auto-imports.d.ts` лишено без змін — `// @ts-nocheck` там ставить `unplugin-auto-import` автоматично.
- Команди перевірки: `bunx tsc -p jsconfig.json --noEmit` (exit 0) та `bunx tsc --noEmit --allowJs --strict … vite.config.js test/happy-dom.preload.js stryker.config.mjs` (exit 0).

## ADR Розширення glob у `.gitignore` для cargo-mutants артефактів

## Context and Problem Statement
`cargo-mutants` при повторному запуску перейменовує попередній результат у `mutants.out.old/`. У `.gitignore` було зафіксовано лише `mutants.out/`, тому `app/src-tauri/mutants.out.old/` залишався untracked і засмічував `git status`.

## Considered Options
* Змінити патерн на `mutants.out*/` (покриває обидва каталоги одним glob)
* Додати окремий рядок `mutants.out.old/`

## Decision Outcome
Chosen option: "Змінити патерн на `mutants.out*/`", because один glob покриває будь-яку кількість ротованих директорій (`mutants.out/`, `mutants.out.old/`, і потенційно майбутні варіанти) без дублювання рядків.

### Consequences
* Good, because `git check-ignore -v app/src-tauri/mutants.out.old/caught.txt` підтвердив, що файл ігнорується рядком `.gitignore:39:mutants.out*/`.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
- Змінений файл: `.gitignore`, рядок 39: `mutants.out/` → `mutants.out*/`
- Пов'язані артефакти: `app/src-tauri/mutants.out.old/{caught.txt,debug.log,diff,lock.json,log,missed.txt,mutants.json,outcomes.json,timeout.txt,unviable.txt}`

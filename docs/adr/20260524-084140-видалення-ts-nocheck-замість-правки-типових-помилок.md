## ADR Видалення `// @ts-nocheck` замість правки типових помилок

## Context and Problem Statement
Робоче дерево містило `// @ts-nocheck` на першому рядку 9 файлів (`app/src/App.test.js`, `app/src/services/auth-store.test.js`, `app/src/views/Login.test.js`, `app/src/i18n/auth-errors.test.js`, `app/src/test-utils/quasar.js`, `app/src/vite-env.d.ts`, `app/vite.config.js`, `app/test/happy-dom.preload.js`, `app/stryker.config.mjs`). `git blame` показав, що ці рядки з'явились у незакомічених змінах. Потрібно було встановити, чи файли справді мають типові помилки, чи це просто пригнічення перевірки.

## Considered Options
* Залишити `// @ts-nocheck` (не чіпати, якщо `tsc` звітує реальні помилки)
* Видалити `// @ts-nocheck` і перевірити `tsc --noEmit`

## Decision Outcome
Chosen option: "Видалити `// @ts-nocheck` і перевірити `tsc --noEmit`", because `bunx tsc -p app/jsconfig.json --noEmit` (TypeScript 6.0.3) повернув exit 0 з 0 помилок після видалення у всіх файлах. `app/jsconfig.json` має `include: ["src/**/*"]` — файли поза `src/` (`vite.config.js`, `test/happy-dom.preload.js`, `stryker.config.mjs`) взагалі не охоплені перевіркою, тобто `@ts-nocheck` там був без ефекту.

### Consequences
* Good, because `app/src/vite-env.d.ts` відповідає канону n-vue (рівно `/// <reference types="vite/client" />`), що вимагає один рядок.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Файл `app/auto-imports.d.ts` залишено без змін — `// @ts-nocheck` там ставить сам плагін `unplugin-auto-import` під час генерації. Перевірка запускалась командою `bunx tsc -p app/jsconfig.json --noEmit`.

## ADR Розширення `.gitignore` для покриття `mutants.out*/`

## Context and Problem Statement
`cargo-mutants` при повторному запуску перейменовує попередній прогін у `mutants.out.old/`. Існуючий `.gitignore` мав запис `mutants.out/`, але не охоплював `mutants.out.old/`, через що ця директорія залишалась untracked.

## Considered Options
* Додати окремий рядок `mutants.out.old/`
* Замінити `mutants.out/` на glob `mutants.out*/`

## Decision Outcome
Chosen option: "Замінити `mutants.out/` на glob `mutants.out*/`", because це покриває будь-які майбутні суфікси, які може продукувати cargo-mutants (`mutants.out/`, `mutants.out.old/`, тощо), одним записом замість переліку.

### Consequences
* Good, because `git check-ignore -v app/src-tauri/mutants.out.old/caught.txt` підтверджує, що директорія тепер ігнорується.
* Bad, because transcript не містить підтверджених негативних наслідків.

## More Information
Зміна у файлі `.gitignore`, рядок 39: `mutants.out/` → `mutants.out*/`. Артефакти cargo-mutants знаходяться у `app/src-tauri/mutants.out.old/` (caught.txt, debug.log, diff, lock.json, log, missed.txt, mutants.json, outcomes.json, timeout.txt, unviable.txt).

## ADR Реалізація List-Unsubscribe (RFC 2369/8058) з sticky bottom bar

## Context and Problem Statement
Застосунок не мав механізму для відписки від розсилок: `GmailMessage` не містив поля `unsubscribe`, в UI не було кнопок дій під листом. Прокрутка до кнопок у довгому листі ускладнена, особливо на мобільних пристроях.

## Considered Options
* Regex-пошук посилань «unsubscribe» у тілі листа
* Парсинг стандартного заголовка `List-Unsubscribe` (RFC 2369/8058)
* Комбо: header-парсинг як основний шлях + body-fallback
* Sticky bottom bar для кнопок дій
* Top toolbar (sticky при скролі)
* Floating Action Button

## Decision Outcome
Chosen option: "Парсинг `List-Unsubscribe` заголовка + sticky bottom bar", because стандартний заголовок доступний у більшості листів-розсилок (особливо після вимог Google/Yahoo 2024), а sticky bottom bar відповідає thumb-zone мобільних пристроїв і є стандартом Apple Mail / Outlook Mobile.

### Consequences
* Good, because transcript фіксує очікувану користь: кнопки завжди доступні без прокрутки; one-click unsubscribe не вимагає відкриття браузера; бонус-команда `gmail_random_newsletter` (q=`has:list-unsubscribe`) дозволяє переглядати лише newsletters.
* Bad, because fallback для листів без `List-Unsubscribe` (body-scan) не реалізований у цій сесії — кнопка «Відписатися» буде disabled для таких листів.

## More Information
Реалізовано: enum `UnsubscribeAction { OneClick{url} | Url{url} | Mailto{to,subject} }` у `app/src-tauri/src/gmail/message.rs`; tauri-команди `gmail_unsubscribe` та `gmail_random_newsletter` у `app/src-tauri/src/gmail/mod.rs`; нові стани `isUnsubscribing`, `unsubscribeErrorKind`, `onlyNewsletters` у `app/src/services/auth-store.js`; sticky bottom bar через `q-page-sticky position="bottom"` + `safe-area-inset-bottom` у `app/src/views/Login.vue`. OneClick надсилає `POST` з body `List-Unsubscribe=One-Click` та `Content-Type: application/x-www-form-urlencoded`; URL/Mailto відкриваються через `tauri-plugin-opener`.

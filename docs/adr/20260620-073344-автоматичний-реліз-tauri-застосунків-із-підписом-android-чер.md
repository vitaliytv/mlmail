## ADR Автоматичний реліз Tauri-застосунків із підписом Android через Infisical OIDC

## Context and Problem Statement

Потрібно налаштувати автоматичний реліз GitHub для Tauri-застосунків mlmail, myshare та task, що дозволяє завантажувати `.dmg` (macOS) і `.apk` (Android) із кожного git-тегу `v*`. Android-збірки потребують підпису keystore, а облікові дані підпису необхідно зберігати безпечно — без прямого зберігання в GitHub Secrets.

## Considered Options

* **GitHub Secrets напряму** — залити `ANDROID_KEYSTORE_BASE64` та інші 4 значення в GitHub Actions Secrets репозиторію.
* **Infisical Universal Auth (client-id + client-secret)** — зберігати Machine Identity credentials у GitHub Secrets, а Android-секрети тягнути з Infisical під час CI.
* **Infisical OIDC Machine Identity** — GA отримує OIDC JWT від GitHub; Infisical йому довіряє і видає Android-секрети. Нуль секретів у GitHub. Саме цей патерн вже використовується в `nitra/ai` через `Infisical/secrets-action@v1.0.16`.

## Decision Outcome

Chosen option: "Infisical OIDC Machine Identity", because цей патерн вже встановлений в `nitra` (workflow `gt-run.yml` в `nitra/ai`), не вимагає жодних секретів у GitHub-репо, а identity `53691c96-17d9-4389-b078-0f77073809ab` вже налаштована в Infisical для проєкту `vitaliytv-kfse`.

### Consequences

* Good, because Android-секрети (`ANDROID_KEYSTORE_BASE64`, `ANDROID_KEYSTORE_PASSWORD`, `ANDROID_KEY_ALIAS`, `ANDROID_KEY_PASSWORD`) зберігаються лише в Infisical (env `main`, path `/mlmail`), і жодного з них немає в GitHub.
* Good, because якщо keystore потрібно ротувати — достатньо оновити Infisical; workflow змінювати не треба.
* Bad, because `tauri android build` у `task` (`nitra/task`) неможливий поки не виконано `tauri android init` і не закомічено `app/src-tauri/gen/android` — тому там наразі лише desktop-only (DMG).

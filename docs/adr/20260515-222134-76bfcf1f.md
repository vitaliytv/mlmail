## ADR Відображення випадкового листа з INBOX на стартовому екрані

**Контекст:** MLMaiL хотів показувати один рандомний лист із INBOX після логіну — для «живого» вигляду стартового екрану. Потрібно було вирішити: який sample-space для вибору, який обсяг тіла показувати, де тримати стан і коли оновлювати лист.

**Рішення/Процедура/Факт:**
- **Sample-space**: `messages.list?labelIds=INBOX&maxResults=100` → `rand::random` індекс. Два HTTP-виклики (`list` + `get`).
- **Body extraction**: `gmail/message.rs` — рекурсивний обхід `payload.parts`; перевага `text/plain` над `text/html`; для HTML-fallback: strip тегів через `regex` + decode HTML-entities через `html-escape`; truncate до 10 000 символів.
- **Стан**: нові ref-и `currentMessage`, `messageErrorKind`, `isMessageLoading` та метод `loadRandomMessage()` в існуючому `auth-store.js` (не окремий composable).
- **Trigger**: кнопка «Показати інший» + авто-виклик після `initialize()`/`login()`.
- **UI**: `<pre class="message-body">` для збереження переносів; `overflow-wrap: anywhere` замість `word-break: break-word`.
- **Нові файли**: `app/src-tauri/src/gmail/message.rs` (DTO + pure functions), розширений `gmail/mod.rs` (helpers + Tauri-команда `gmail_random_message`).
- **i18n**: новий ключ `Empty` → «Скринька порожня.»; `GmailError::Empty` у Rust.

**Обґрунтування:** 100 останніх — достатньо для UX «живого» листа без N-пагінаційних запитів; plain-text body виключає необхідність DOMPurify/iframe-ізоляції; тримати стан в `auth-store.js` уникає зайвої залежності в `Login.vue`; окремий `message.rs` робить pure-functions testable без mockito.

**Розглянуті альтернативи:**
- Random по всьому INBOX через pageToken-пагінацію — відкинуто (50+ запитів для великої скриньки).
- Повний HTML-рендер через DOMPurify + iframe — відкинуто як окрема ітерація.
- Тільки snippet (~200 символів без body) — відкинуто на користь full `text/plain`.
- Окремий composable для message-стану — відкинуто (YAGNI, один composable простіший).

**Зачіпає:** `app/src-tauri/src/gmail/message.rs` (новий), `app/src-tauri/src/gmail/mod.rs`, `app/src-tauri/src/gmail/error.rs`, `app/src-tauri/src/lib.rs`, `app/src/services/auth-store.js`, `app/src/views/Login.vue`, `app/src/i18n/auth-errors.js`, `app/src-tauri/Cargo.toml` (нові deps: `regex`, `html-escape`).

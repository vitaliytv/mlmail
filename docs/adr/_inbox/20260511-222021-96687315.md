## ADR Google OAuth авторизація MLMaiL: архітектура та ключові рішення

**Контекст:** MLMaiL потребує авторизації через Google (scope `gmail.modify`) одночасно для macOS Desktop (Tauri) і Android, де механізми redirect та безпечного зберігання токенів принципово різні між платформами.

**Рішення/Процедура/Факт:**
- **Redirect-механізм:** platform-native — loopback 127.0.0.1:RANDOM_PORT на macOS (через `tauri-plugin-opener`), Credential Manager + AuthorizationClient через JNI-міст на Android.
- **Android SDK:** Credential Manager + Authorization API (не AppAuth-Android) — обрано свідомо, незважаючи на більшу складність JNI.
- **Token storage:** Keychain (macOS) + EncryptedSharedPreferences (Android) під ключами `mlmail.google.refresh_token` і `mlmail.google.email`; access token — виключно in-memory у `AuthState`.
- **Поділ відповідальності:** Rust тримає всі токени; Vue-фронтенд отримує лише `email` після логіну і кличе `auth_get_access_token` перед кожним HTTP-запитом — без жодного токена у JS-пам'яті.
- **Нові файли:** `app/src-tauri/src/auth/` (mod.rs, state.rs, pkce.rs, token_exchange.rs, storage/{mod,macos,android}.rs, flow/{macos,android}.rs); `app/src/services/auth-store.js`; `app/src/views/Login.vue`. Команда `greet` видаляється.
- **Tauri-команди:** `auth_start_login`, `auth_get_access_token`, `auth_is_authenticated`, `auth_current_email`, `auth_logout`. `AuthError` серіалізується як discriminated union `{kind, message}`.
- **Cold start:** `setup()` зчитує refresh_token зі сховища; перший `get_access_token` тригерить refresh. `invalid_grant` → очищення сховища + `AuthError::ReauthRequired` → Login screen.

**Обґрунтування:** Тримати всі токени у Rust (не у webview) мінімізує attack surface: JS-контекст ніколи не має refresh token і не може його витікати через XSS або devtools. Platform-native SDK замість loopback-only підходу обраний для відповідності вимогам Google щодо Android-застосунків і кращого UX (системний bottom-sheet замість браузерного redirect).

**Розглянуті альтернативи:** loopback + custom scheme (відхилено: не оптимально для Android), AppAuth-Android (відхилено: користувач явно обрав Credential Manager API), Pinia для Auth Store (відхилено: обрано модульний singleton-composable без Pinia для мінімізму ітерації), зберігання токенів у webview localStorage (відхилено: принциповий security вибір).

**Зачіпає:** `app/src-tauri/src/auth/` (новий модуль), `app/src-tauri/src/lib.rs` (реєстрація state + invoke_handler), `app/src-tauri/Cargo.toml` (нові залежності: reqwest, keyring або platform-specific crates), `app/src/services/auth-store.js` (новий), `app/src/views/Login.vue` (новий), `app/src/App.vue` (видалення greet-демо), `docs/ci4/` (Auth Component і Auth Store переходять з `planned` у `active`).

## ADR Архітектура Google OAuth з розділенням публічних змінних і секретів у MLMaiL

**Контекст:** MLMaiL (Tauri 2 + Vue 3, macOS/Android) потребує Google OAuth авторизації через Authorization Code + PKCE flow. З'ясувалось, що Google вимагає `client_secret` у Desktop OAuth client навіть при використанні PKCE, а compile-time `option_env!` для client IDs призводив до помилки `invalid_client` якщо `cargo` запускався без env vars у shell.

**Рішення/Процедура/Факт:**
- `auth/config.rs` — runtime читання через `dotenvy` + `OnceLock`, замість compile-time `option_env!`. При ініціалізації завантажуються два файли: `.env` (публічні IDs — у git) і `.env.secret` (секрети — у gitignore), причому другий не перезаписує перший.
- `auth/token_exchange.rs` — `exchange_code_at` і `exchange_refresh_at` приймають `Option<&str>` для `client_secret`; Desktop flow передає `Some(secret)`, Android flow — `None`.
- `auth/flow/macos.rs` — loopback HTTP сервер + `tauri-plugin-opener` + Keychain (`keyring` crate); передає `client_secret` у token exchange.
- `auth/flow/android.rs` — Tauri 2 mobile plugin → Kotlin `MlmailAuthPlugin`; `client_secret` відсутній (SHA-1 fingerprint автентифікує).
- `.env.example` містить три публічні IDs; `.env.secret.example` — шаблон для `MLMAIL_GOOGLE_DESKTOP_CLIENT_SECRET`.
- `.gitignore` — `app/src-tauri/.env.secret` ігнорується, `app/src-tauri/.env` трекається.

**Обґрунтування:** Client IDs для native apps є публічними ідентифікаторами (видно з `strings mlmail` і в OAuth traffic), тому їх безпечно тримати у git приватного репо. `client_secret` для Desktop client — умовний "секрет" (Google сам зазначає, що він не справжній секрет), але за конвенцією він не трекується. Runtime читання через `dotenvy` усуває необхідність `cargo clean` після зміни `.env` і зняло клас помилок де бінарник містив placeholder `REPLACE_ME_...`.

**Розглянуті альтернативи:** compile-time `option_env!` (відкинуто — не перезавантажується без `cargo clean`, дає `invalid_client` якщо env не було у shell при збірці); один `.env` для всього (відкинуто — змішує публічне з секретним, ускладнює рішення про gitignore).

**Зачіпає:** `app/src-tauri/src/auth/config.rs`, `app/src-tauri/src/auth/token_exchange.rs`, `app/src-tauri/src/auth/flow/macos.rs`, `app/src-tauri/src/auth/flow/android.rs`, `app/src-tauri/src/auth/mod.rs`, `app/src-tauri/.env`, `app/src-tauri/.env.secret` (gitignored), `app/src-tauri/.env.example`, `app/src-tauri/.env.secret.example`, `.gitignore`.

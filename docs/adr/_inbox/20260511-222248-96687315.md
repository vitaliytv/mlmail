## ADR Google OAuth 2.0 + PKCE авторизація в MLMaiL (macOS і Android)

**Контекст:** MLMaiL потребує авторизації через Google (scope `gmail.modify`) на двох платформах — macOS Desktop і Android. C4-модель вже резервувала місця `Auth Component` та `Auth Store` у статусі `planned`; відкриті ADR-питання стосувалися вибору redirect-механізму, SDK і сховища токенів.

**Рішення/Процедура/Факт:**
Реалізується у цій ітерації **лише OAuth flow + зберігання токенів** (Gmail API і роутер — наступні ітерації).

*Компоненти:*

- `app/src/views/Login.vue` — мінімальний екран із кнопкою "Sign in with Google".
- `app/src/services/auth-store.js` — singleton Vue composable; тримає лише `isAuthenticated`, `email` (не токени), проксіює Tauri-команди.
- `app/src-tauri/src/auth/` — новий Rust-модуль: `mod.rs` (5 Tauri-команд), `state.rs` (`AuthState` in-memory), `pkce.rs` (PKCE-пара SHA-256), `token_exchange.rs` (POST до `oauth2.googleapis.com/token` через `reqwest`+rustls), `id_token.rs` (JWT payload → email без верифікації підпису), `storage/` (trait + macOS Keychain / Android EncryptedSharedPreferences).

*Ключові потоки:*

- macOS: PKCE → loopback `127.0.0.1:RANDOM_PORT` → `tauri-plugin-opener` відкриває браузер → auth code → POST token exchange.
- Android: PKCE → JNI до Kotlin `AuthHelper.signInAndAuthorize()` (Credential Manager + AuthorizationClient) → auth code → POST token exchange.
- Cold start: `setup()` зчитує refresh token зі сховища; перший виклик `auth_get_access_token` тригерить тихий refresh.
- Refresh expiry guard: 30-секундний запас перед `expires_at` у `is_access_token_fresh()`.

*Tauri-команди (реєструються у `lib.rs`):* `auth_start_login`, `auth_get_access_token`, `auth_is_authenticated`, `auth_current_email`, `auth_logout`.

*Серіалізація помилок:* `AuthError` як discriminated union `{kind, message}` — frontend фільтрує `kind === 'ReauthRequired'` без парсингу рядка.

*Нові Cargo-залежності:* `reqwest 0.12` (rustls-tls, json), `base64 0.22`, `rand 0.9`, `sha2 0.10`, `thiserror`, `serde_json`.

**Обґрунтування:**

- **Platform-native замість loopback-only:** Android не підтримує надійний loopback redirect, а Credential Manager є рекомендованим Google API для нових Android-застосунків з 2023 р.
- **Credential Manager + Authorization API (не AppAuth):** забезпечує bottom-sheet UX із Zero Trust; AppAuth — застаріла альтернатива.
- **Keychain / EncryptedSharedPreferences:** OS-рівневе шифрування без власного ключа; аналог у `tauri-plugin-stronghold` додає складність без переваг для MVP.
- **Rust тримає всі токени, JS отримує лише access token на запит:** refresh token ніколи не потрапляє у WebView JS-середовище — мінімальна поверхня атаки.
- **rustls замість OpenSSL:** спрощений Android cross-compile build chain.
- **JWT payload без верифікації підпису:** токен щойно отриманий по HTTPS від `oauth2.googleapis.com`; email потрібен лише для UI, не для authorization — верифікація була б YAGNI.
- **Видалення `greet` команди:** C4-документ вже фіксував її як тимчасову.

**Розглянуті альтернативи:**

- `tauri-plugin-oauth` (loopback-only) — відхилено: не вирішує Android.
- AppAuth-Android — відхилено на користь Credential Manager (новіший стандарт Google).
- `tauri-plugin-stronghold` — відхилено: зайва складність для MVP; OS keystore достатньо.
- Pinia замість singleton composable — відхилено: Auth State не потребує devtools/SSR, додає залежність.

**Зачіпає:** `app/src/App.vue`, `app/src/views/Login.vue` (новий), `app/src/services/auth-store.js` (новий), `app/src-tauri/src/lib.rs`, `app/src-tauri/src/auth/` (новий модуль цілком), `app/src-tauri/Cargo.toml`, `tauri.conf.json` (permissions для opener + platform-specific deep link scheme).

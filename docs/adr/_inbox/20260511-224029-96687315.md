## ADR Google OAuth 2.0 Авторизація для MLMaiL: платформо-нативні потоки з Rust-owned токенами

**Контекст:** MLMaiL (Tauri 2 + Vue 3, macOS + Android) потребує Google OAuth 2.0 авторизації з доступом до `gmail.modify`; архітектура C4-моделі описувала `Auth Component` і `Auth Store` як `planned`, але конкретний механізм redirect, сховище токенів і межа відповідальності Rust/Vue залишалися відкритими ADR-питаннями.

**Рішення/Процедура/Факт:** Реалізувати авторизацію у двох гілках: macOS — Authorization Code + PKCE з loopback HTTP server (`tokio::net::TcpListener` на випадковому порті) і `tauri-plugin-opener` для браузера; Android — Credential Manager (`GetGoogleIdOption` з Web client ID) + Authorization API (`requestOfflineAccess(forceCodeForRefreshToken=true)`) через JNI-міст до Kotlin. Rust (`app/src-tauri/src/auth/`) зберігає refresh token у Keychain (macOS) або EncryptedSharedPreferences (Android) через trait `RefreshTokenStorage`; access token — лише in-memory у `AuthState`. Vue (`auth-store.js`, `Login.vue`) отримує виключно `{ email, isAuthenticated }` — жодних токенів у JS. Нові Tauri-команди: `auth_start_login`, `auth_get_access_token`, `auth_is_authenticated`, `auth_current_email`, `auth_logout`. Специфікація: `docs/superpowers/specs/2026-05-11-google-auth-design.md`; план (10 задач): `docs/superpowers/plans/2026-05-11-google-auth-plan.md`.

**Обґрунтування:** Loopback + PKCE на macOS відповідає RFC 8252 (Desktop app без client_secret); для Android Credential Manager — офіційно рекомендована Google заміна deprecated `GoogleSignIn`; `forceCodeForRefreshToken=true` є обов'язковим для отримання `serverAuthCode`. Rust-owned токени виключають витік refresh_token у JS-пам'ять або DevTools. `rustls-tls` у `reqwest` спрощує Android build chain (без OpenSSL). `keyring` crate з `apple-native` feature уникає D-Bus залежностей. `AuthError` серіалізується як discriminated union `{ kind, message }` — Vue може перевіряти `error.kind === 'ReauthRequired'` без парсингу рядка.

**Розглянуті альтернативи:**

- `tauri-plugin-oauth` (community plugin для loopback) — відхилено на користь прямої tokio/hyper реалізації (~80 рядків, повний контроль);
- AppAuth-Android — відхилено на користь Credential Manager + Authorization API (офіційний, нова платформа Google);
- `tauri-plugin-deep-link` (custom URI scheme на Android) — не обговорювався детально після вибору native SDK;
- Pinia для Auth Store — відхилено на користь module-level singleton composable (простіше, лише один Auth Store на застосунок);
- `WEB_CLIENT_ID`/`DESKTOP_CLIENT_ID` у `.env` — відхилено, зберігаємо у `google_client_ids.rs` (у `.gitignore`) з `.example`-шаблоном у git.

**Зачіпає:** `app/src-tauri/src/auth/` (новий модуль цілком), `app/src-tauri/src/lib.rs`, `app/src-tauri/Cargo.toml`, `app/src-tauri/gen/android/app/build.gradle`, `app/src-tauri/gen/android/.../auth/*.kt` (новий Kotlin модуль), `app/src/views/Login.vue` (новий), `app/src/services/auth-store.js` (новий), `app/src/App.vue`; Google Cloud Console: Desktop app client, Web application client, Android client.

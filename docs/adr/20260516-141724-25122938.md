## ADR Виправлення lint-ланцюжка `bun run lint` у mlmail

**Контекст:** Після додавання нових workflow і конфігів у попередній сесії (`/n-fix`) кілька інструментів в `bun run lint` стали падати: knip знаходив невикористані залежності/бінарники, v8r намагався валідувати згенеровані Rust/Tauri файли, cspell не знав UA-транслітерацій, shellcheck знайшов `SC2086` у workflow.

**Рішення/Процедура/Факт:**
- `app/package.json`: видалено залежність `@tauri-apps/plugin-opener` (JS-сторона не використовувала її; Rust-сторона `tauri-plugin-opener` у `Cargo.toml` залишилась)
- `knip.json`: у `ignore` додано `app/src-tauri/target/**`; у `ignoreDependencies` — `@nitra/stylelint-config`; у `ignoreBinaries` — `stylelint` (викликається через `npx` за правилом `n-style-lint.mdc`, тому knip не бачить прямого bun-виклику)
- `.cspell.json`: виправлено справжню одруківку `rендер` (латинська `r`) → `рендер` в ADR-файлі; до `words` додано 13 легітимних UA-транслітерацій технічного жаргону (`фічі`, `хелпери`, `стейтів`, `скіл`, тощо)
- `.v8rignore`: переписано патерни у gitignore-style з trailing-slash; додано `app/src-tauri/target/`, `app/src-tauri/gen/`, `app/src-tauri/capabilities/`, `app/src-tauri/tauri.conf.json` (всі без записів у Schema Store)
- `.github/workflows/git-ai.yml`: `$GITHUB_PATH` → `"$GITHUB_PATH"` (SC2086 shellcheck)
- Фінальний результат: `bun run lint` exit 0

**Обґрунтування:** Всі виправлення — реальні проблеми, не заглушки: невикористана залежність видалена, одруківка виправлена в джерелі, Tauri-генеровані файли явно виключені з v8r бо їх схеми Schema Store не знає, UA-транслітерації легітимні й потрібні в ADR-документах.

**Розглянуті альтернативи:** `@tauri-apps/plugin-opener` можна було перенести в `knip.json -> ignoreDependencies` замість видалення, але оскільки JS-код не посилається на нього, видалення правильніше. Для v8r розглядалися `**`-префіксні патерни — вони не спрацьовували (v8r використовує gitignore-стиль без `**` для директорій).

**Зачіпає:** `app/package.json`, `bun.lock`, `knip.json`, `.cspell.json`, `.v8rignore`, `.github/workflows/git-ai.yml`, `docs/adr/20260516-135705-76bfcf1f.md`

# C4-модель проєкту MLMaiL

C4-модель проєкту MLMaiL — джерело істини про архітектуру застосунку MLMaiL (особистий
AI-асистент для обробки Gmail з голосовим саммері). Документ слідує принципам із
`.cursor/rules/n-ci4.mdc`: Markdown — джерело істини, Spec-as-Source, Rebuild Test,
контекстна незалежність розділів.

## Що таке MLMaiL

MLMaiL — крос-платформений застосунок (macOS desktop і Android), який:

- авторизує користувача через Google OAuth 2.0;
- читає вхідні листи з Gmail;
- готує AI-саммері кожного листа;
- озвучує саммері (TTS);
- пропонує дію над листом: видалити, видалити та додати фільтр Gmail,
  зафіксувати у локальній замітці `home/`, зафіксувати у локальній замітці `work/`;
- на завершення складає чернетку відповіді з пропозицією користувачу.

## Рівні C4-моделі

C4-модель MLMaiL складається з чотирьох рівнів і двох допоміжних файлів:

- [01-context.md](01-context.md) — System Context. Хто користувач застосунку MLMaiL,
  з якими зовнішніми системами він спілкується (Google Identity, Gmail API,
  LLM-провайдер, TTS-провайдер).
- [02-containers.md](02-containers.md) — Containers. Які виконувані одиниці
  складають застосунок MLMaiL: Vue 3 frontend, Tauri-обгортка з Rust-бекендом,
  локальне сховище нотаток.
- [03-components.md](03-components.md) — Components. Які логічні компоненти
  живуть всередині Vue 3 frontend і Rust-бекенду застосунку MLMaiL.
- [04-code.md](04-code.md) — Code. Ключові класи, функції, файли застосунку
  MLMaiL: Tauri-команди, Vue-компоненти, конфігурація.
- [decisions.md](decisions.md) — зведений вплив ADR на C4-модель MLMaiL.

## Статус проєкту MLMaiL

На дату написання документації репозиторій MLMaiL містить **стартовий каркас**:
Tauri 2 + Vue 3 шаблон з демо-командою `greet`. Більшість компонентів, описаних
у C4-моделі MLMaiL, явно позначені як `planned` — це навмисно: документація
описує цільову архітектуру MLMaiL, до якої агент має код **дотягнути**, а не
лише поточний стартовий стан. Розділ `Поточний стан` усередині кожного рівня
відокремлює реалізоване від запланованого.

## Як читати документацію MLMaiL

1. Перед будь-якою зміною коду відкривай той рівень C4-моделі MLMaiL, до якого
   стосується редагований файл (фронтенд → 03-components.md, Tauri-команда →
   03-components.md і 04-code.md, новий зовнішній сервіс → 01-context.md і
   02-containers.md).
2. Якщо зміна **впливає** на C4-модель MLMaiL — оновлюй відповідний рівень у
   тому ж PR (правило `Оновлення синхронно зі змінами` з `.cursor/rules/n-ci4.mdc`).
3. Якщо рішення суттєве — створюй ADR у `docs/adr/` і додавай посилання у
   [decisions.md](decisions.md).

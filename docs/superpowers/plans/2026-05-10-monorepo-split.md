# Monorepo Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure the single-package Tauri+Vue project into a bun monorepo with one workspace `app/` containing all Tauri-related code, while configs and tooling stay at the repo root.

**Architecture:** Bun workspaces with a single workspace `app/`. Root `package.json` declares `workspaces: ["app"]`, holds only shared dev dependencies (`@nitra/cursor`), and has no scripts. The `app/` workspace owns the frontend (`src/`, `public/`, `index.html`, `vite.config.js`), the Tauri Rust app (`src-tauri/`), all Vite/Vue/Tauri dependencies, and dev/build/preview/tauri scripts.

**Tech Stack:** Bun (workspaces + linker=hoisted), Vite 8, Vue 3, @vitejs/plugin-vue 6, Tauri 2, Cargo.

**Spec:** [`docs/superpowers/specs/2026-05-10-monorepo-split-design.md`](../specs/2026-05-10-monorepo-split-design.md)

# n-coverage-fix — ітеративний запис тестів для вижилих мутантів

## Мета

Запустити `/n-coverage`, прочитати `reports/stryker/survived.md`, для кожного вижилого мутанта
(по одному) дописати один тест, перевірити що проходить, потім перезапустити `/n-coverage`
і повторювати поки кількість вижилих не перестане зменшуватись (конвергенція).

## Передумови

- CWD — корінь проєкту де є `package.json` зі скриптами `coverage` і `test`
- Stryker сконфігурований (`stryker.config.mjs`)

## Workflow

### Step 1: Запустити n-coverage

Виклич skill `n-coverage` (через Skill tool з `skill: "n-coverage"`).

Прочитай `reports/stryker/survived.md` (або `app/reports/stryker/survived.md`).
Підрахуй кількість рядків даних (не рахуючи заголовок і header-рядок таблиці).
Збережи як `previousCount`. Збережи початкове значення як `initialCount`.

Якщо файл відсутній або рядків даних 0 → виведи `✅ Конвергенція: вижилих мутантів немає` і зупинись.

### Step 2: Для кожного мутанта (по одному)

Обробляй рядки з таблиці `survived.md` послідовно.

#### 2a. Зчитати контекст мутанта

З рядка таблиці витягни: `Файл`, `Рядок`, `Тип мутації`, `Оригінал`, `Заміна Stryker`,
`Причина виживання`, `Що тестувати`.

- Прочитай source файл (колонка `Файл`), витягни ±15 рядків навколо `Рядок`
- Знайди test файл за конвенцією:
  1. `src/foo.js` → `src/foo.test.js`
  2. `src/foo.js` → `test/foo.test.js`
  3. `src/foo.js` → `src/__tests__/foo.test.js`
  4. Якщо не знайдено → `grep -r "$(basename <file> .js)" --include="*.test.*" -l`
- Прочитай існуючий test файл (якщо є)

#### 2b. Написати один тест

Використай:
- Source контекст (оригінальний код на мутованому рядку ± оточення)
- `Оригінал` → `Заміна Stryker` (що конкретно змінено)
- `Що тестувати` з survived.md (LLM-рекомендація)
- Формат, мова, framework існуючих тестів

Напиши ОДИН test case що:
1. Тестує поведінку оригінального коду (`Оригінал`)
2. Провалиться якщо `Заміна Stryker` буде застосована
3. Відповідає стилю існуючих тестів (тій самій мові, опису, структурі)

Додай тест до test файлу. Не змінюй і не видаляй наявні тести.

#### 2c. Перевірити що тест проходить

Знайди тест-команду: читай `package.json#scripts.test`.

```bash
bun test <testFile>
```

- Якщо проходить → переходь до наступного мутанта
- Якщо падає → виправ тест (максимум 2 спроби: перечитай помилку, виправ)
- Якщо після 2 спроб все ще падає → пропусти мутант, додай в test файл:
  ```js
  // TODO(n-coverage-fix): not fixed — <mutatorName> line <line> — <replacement>
  ```

### Step 3: Перевірити конвергенцію

Після обробки всіх мутантів → виклич skill `n-coverage` ще раз (Skill tool, `skill: "n-coverage"`).

Прочитай новий `survived.md`, підрахуй рядки → `newCount`.

**Якщо `newCount < previousCount`:**
- Встанови `previousCount = newCount`
- Поверни до Step 2 (нова ітерація з оновленим survived.md)

**Якщо `newCount === previousCount` (конвергенція):**

```
✅ Конвергенція досягнута
   Було вижилих: <initialCount>
   Залишилось:   <newCount>
   Файл:         reports/stryker/survived.md
```

Зупинись.
